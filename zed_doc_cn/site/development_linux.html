为 Linux 构建 Zed

代码仓库

克隆 Zed 代码仓库。

依赖项

  • 安装 rustup

  • 安装必要的系统库：

    script/linux

    如果您更倾向于手动安装系统库，可以在 script/linux 文件中找到所需软件包的列表。

后端依赖项（可选）

如果您希望使用本地协作服务器开发 Zed 协作功能，请参阅：本地协作 文档。

从源码构建

安装依赖项后，您可以使用 Cargo 构建 Zed。

构建调试版本的编辑器：

cargo run

运行测试：

cargo test --workspace

在发布模式下，主要用户界面是 cli crate。您可以在开发中运行：

cargo run -p cli

安装开发版本

您可以使用以下命令在计算机上安装本地构建：

./script/install-linux

这将以发布模式构建 zed 和 cli，并将它们放置在 ~/.local/bin/zed，同时将 .desktop 文件安装到 ~/.local/share。

Wayland 和 X11

Zed 支持 X11 和 Wayland。默认情况下，我们选择在运行时能找到的任一个。如果您在 Wayland 上运行并希望使用 X11 模式，请使用环境变量 WAYLAND_DISPLAY=''。

Zed 打包说明

感谢您承担 Zed 的打包工作！

技术要求

Zed 有两个主要的二进制文件：

  • 您需要构建 crates/cli 并使其二进制文件在 $PATH 中以 zed 名称可用。
  • 您需要构建 crates/zed 并将其放置在 $PATH/to/cli/../../libexec/zed-editor。例如，如果您要将 cli 放在 ~/.local/bin/zed，请将 zed 放在 ~/.local/libexec/zed-editor。由于一些 Linux 发行版（特别是 Arch）不建议使用 libexec，您也可以将此二进制文件放在 $PATH/to/cli/../../lib/zed/zed-editor（例如 ~/.local/lib/zed/zed-editor）。
  • 如果您要提供 .desktop 文件，可以在 crates/zed/resources/zed.desktop.in 中找到模板，并使用 envsubst 填充所需的值。此文件还应重命名为 $APP_ID.desktop，以便文件遵循 FreeDesktop 标准。您还应该使此桌面文件可执行（chmod 755）。
  • 您需要确保安装了必要的库。您可以通过在系统上检查构建的二进制文件来获取当前列表。
  • 有关完整构建脚本的示例，请参阅 script/bundle-linux。
  • 您可以通过使用环境变量 ZED_UPDATE_EXPLANATION 构建（或运行）Zed 来禁用 Zed 的自动更新并为尝试手动更新 Zed 的用户提供说明。例如：ZED_UPDATE_EXPLANATION="请使用 flatpak 更新 zed。"。
  • 确保将 crates/zed/RELEASE_CHANNEL 文件的内容更新为 'nightly'、'preview' 或 'stable'，不带换行符。这将使 Zed 使用凭据管理器来记住用户的登录信息。

其他注意事项

在 Zed，我们的首要任务是快速发展并为用户带来最新技术。我们长期以来一直对缓慢、过时或难以配置的软件感到沮丧，因此我们根据这些需求构建了我们的编辑器。

但是，我们意识到许多发行版有其他优先级。我们希望与所有人合作，将 Zed 带到他们喜欢的平台上。但还有很长的路要走：

  • Zed 是一个快速发展的早期阶段项目。我们通常每周发布 2-3 个构建版本来修复用户报告的问题并发布主要功能。
  • 在 Linux 系统上可能存在其他几个 zed 二进制文件（1，2）。如果您因为这些问题想要重命名我们的 CLI 二进制文件，我们建议使用 zedit、zeditor 或 zed-cli。
  • Zed 会像 rustup/rbenv/pyenv 等工具一样自动安装正确版本的常见开发工具。我们理解这是有争议的，请参阅此处。
  • 我们允许用户在本地和从 zed-industries/extensions 安装扩展。这些扩展可能会根据需要安装进一步的工具，例如语言服务器。从长远来看，我们希望使这更安全，请参阅此处。
  • Zed 默认连接到多个在线服务（AI、遥测、协作）。用户可以通过 zed 设置或修补我们的默认设置文件来禁用 AI 和我们的遥测。
  • 由于上述问题，zed 目前与沙箱不兼容，请参阅此处

Flatpak

Zed 当前的 Flatpak 集成在启动时会退出沙箱。依赖 Flatpak 沙箱功能的工作流可能无法按预期工作。

要在本地构建和安装 Flatpak 包，请按照以下步骤操作：

  1. 按照此处的说明为您的发行版安装 Flatpak。
  2. 运行 script/flatpak/deps 脚本来安装所需的依赖项。
  3. 运行 script/flatpak/bundle-flatpak。
  4. 现在包已安装并在 target/release/{app-id}.flatpak 处有一个可用的包。

内存性能分析

heaptrack 对于诊断内存泄漏非常有用。安装它：

$ sudo apt install heaptrack heaptrack-gui
$ cargo install cargo-heaptrack

然后，要在附加了性能分析器的情况下构建和运行 Zed：

$ cargo heaptrack -b zed

当退出此 zed 实例时，终端输出将包含一个命令来运行 heaptrack_interpret，将 *.raw.zst 文件转换为 *.zst 文件，然后可以将其传递给 heaptrack_gui 进行查看。

Cargo 报告依赖项使用不稳定功能的错误

尝试运行 cargo clean 和 cargo build。
