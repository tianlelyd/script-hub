Git

Zed 目前提供了一系列基础的 Git 功能，未来将支持更高级的功能，如冲突解决工具、逐行暂存等。

以下是当前支持的所有功能概览：

  • 提交
  • 暂存、推送、拉取和获取
  • 项目差异：所有更改的多缓冲区视图
  • 缓冲区和编辑器滚动条中的差异指示器
  • 编辑器中未暂存更改的内联差异切换和恢复
  • 项目面板中的 Git 状态
  • 分支创建和切换
  • Git 责任查看

Git 面板

Git 面板为您提供工作树状态和 Git 暂存区的鸟瞰视图。

您可以使用 git panel: toggle focus 打开 Git 面板，或点击状态栏中的 Git 图标。

在面板中，您可以一目了然地看到项目的状态——哪个仓库和分支处于活动状态、哪些文件已更改以及每个文件的当前暂存状态。

Zed 会监控您的仓库，因此您在命令行中所做的更改会立即反映出来。

项目差异

您可以通过打开项目差异（ctrl-g d|ctrl-g d）来查看 Zed 中 Git 捕获的所有更改，可通过命令面板或 Git 面板中的 git: diff 操作访问。

项目差异中显示的所有更改的行为与任何其他多缓冲区完全相同：它们都是文件的可编辑摘录。

您可以通过点击选项卡栏上的按钮或其相应的键绑定来暂存或取消暂存每个块以及整个文件。

获取、推送和拉取

在 Zed 中，您可以通过 Git 面板上的按钮或通过命令面板查找相应的操作来从 Git 仓库中获取、推送或拉取：git: fetch、git: push 和 git: pull。

暂存工作流程

Zed 有两种主要的暂存工作流程，使用项目差异或直接使用面板。

Using the Project Diff

In the Project Diff view, you can focus on each hunk and stage them individually by clicking on the tab bar buttons or via the keybindings git: stage and next (cmd-y|alt-y).

Similarly, stage all hunks at the same time with the git: stage all (cmd-ctrl-y|ctrl-space) keybinding and then immediately commit with git: commit (cmd-enter|ctrl-enter).

使用 Git 面板

在面板中，您可以简单地输入提交信息并点击提交按钮，或使用 git: commit。这将自动暂存所有被跟踪的文件（在条目的复选框中用 [·] 表示）并提交它们。

可以使用每个单独条目的复选框来暂存条目。可以使用面板顶部的按钮或 git: stage all 暂存所有更改。

提交

Zed 提供两个提交文本区域：

  1. 第一个位于 Git 面板的底部。按下 cmd-enter|ctrl-enter 会立即提交所有已暂存的更改。
  2. 第二个可通过 git: expand commit editor 操作访问，或在 Git 面板提交文本区域获得焦点时按下 shift-escape|。

撤销提交

在 Zed 中提交后，您会在 Git 面板中看到提交文本区域下方的一个栏，它将显示最近提交的提交。 在那里，您可以使用“撤销提交”按钮，它执行 git reset HEADˆ--soft 命令。

Git 中的 AI 支持

Zed 目前支持由 LLM 驱动的提交信息生成。 您可以通过在 Git 面板中聚焦在信息编辑器上，然后点击左下角的铅笔图标，或使用 git: generate commit message（alt-tab|alt-l）键绑定来请求 AI 生成提交信息。

请注意，您需要配置 LLM 提供商。访问 AI 配置页面以了解如何操作。

您可以通过提供 commit_message_model 代理设置来指定您偶爱的模型。有关更多信息，请参阅特定功能模型。

{
  "agent": {
    "version": "2",
    "commit_message_model": {
      "provider": "anthropic",
      "model": "claude-3-5-haiku"
    }
  }
}

未来可能会提供更高级的 AI 与 Git 功能的集成。

Git 集成

Zed 与流行的 Git 托管服务集成，以确保 Git 提交哈希值和对问题、拉取请求和合并请求的引用成为可点击的链接。

Zed 目前支持链接到以下托管版本： GitHub、 GitLab、 Bitbucket、 SourceHut 和 Codeberg。

Zed 还具有复制永久链接功能，可以创建指向您的 Git 托管服务上代码片段的永久链接。 这些链接对于共享特定提交中文件的特定行或行范围很有用。 可以通过命令面板（搜索 permalink）、 通过为 editor::CopyPermalinkToLine 或 editor::OpenPermalinkToLine 操作创建自定义键绑定， 或简单地在编辑器中选择行后右键点击并选择 Copy Permalink 来触发此操作。

差异块键盘快捷键

在查看有更改的文件时，Zed 显示可以展开或折叠以进行详细审查的差异块：

  • 展开所有差异块：editor: expand all diff hunks（cmd-"|ctrl-"）
  • 折叠所有差异块：按 Escape 键（绑定到 editor: cancel）
  • 切换选中的差异块：editor: toggle selected diff hunks（cmd-'|ctrl-'）
  • 在块之间导航：editor: go to hunk 和 editor: go to previous hunk

提示：Escape 键是折叠所有展开的差异块并返回更改概览的最快方式。

Git CLI 配置

如果您希望在从命令行提交时也使用 Zed 作为 git 提交信息编辑器，您可以使用 zed --wait：

git config --global core.editor "zed --wait"

或者将以下内容添加到您的 shell 环境中（在 ~/.zshrc、~/.bashrc 等）：

export GIT_EDITOR="zed --wait"
