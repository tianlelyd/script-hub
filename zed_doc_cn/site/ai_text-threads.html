文本线程

概览

Agent 面板中的文本线程功能与其他编辑器类似。你可以使用自定义快捷键和多光标操作，允许在编码和与语言模型讨论之间无缝切换。

然而，文本线程与其他编辑器的不同之处在于包含了消息块。这些块充当文本容器，对应于上下文中的不同角色。这些角色包括：

  • You（你）
  • Assistant（助手）
  • System（系统）

首先，在 You 块中输入消息。

当你输入时，所选模型的剩余令牌数会实时更新。

从编辑器插入文本很简单，只需高亮显示文本并运行 agent: quote selection（cmd->|ctrl->）；如果是代码，Zed 会自动将其包装在代码围栏块中。

要提交消息，使用 cmd-enter|ctrl-enter（assistant: assist）。与普通线程不同（按 enter 就会提交消息），在文本线程中，我们的目标是让它感觉尽可能接近常规编辑器。因此，按 enter|enter 只是插入新行。

提交消息后，响应将在下方的 Assistant 消息块中流式显示。

可以在任何时候按 escape 键取消响应流。如果你早期意识到响应不是你想要的，这很有用。

如果你想随时开始新对话，可以按 cmd-n|ctrl-n 或使用面板左上角汉堡菜单中的 New Chat 选项。

简单的对话往来在文本线程中运行良好。然而，有时你可能想要修改对话中之前的文本，将其引导到不同的方向。

编辑文本线程

文本线程为你提供了控制上下文的灵活性。 你可以自由编辑任何之前的文本，包括大语言模型的响应。 如果你想完全删除一个消息块，只需将光标放在块的开头并使用 delete 键。 典型的工作流程可能涉及在整个上下文中进行编辑和调整，以完善你的询问或提供额外信息。以下是一个示例：

  1. 在 You 块中写入文本。
  2. 使用 cmd-enter|ctrl-enter 提交消息。
  3. 收到不符合预期的 Assistant 响应。
  4. 使用 escape 取消响应。
  5. 清除 Assistant 消息块的内容并完全删除该块。
  6. 向原始消息添加额外的上下文。
  7. 使用 cmd-enter|ctrl-enter 提交消息。

能够编辑之前的消息让你可以控制令牌的使用方式。你不需要开始新聊天来纠正错误或添加额外信息，也不必通过提交后续更正来浪费令牌。

注意：在语言模型的上下文中，编辑过去消息的行为通常被称为"重写历史"。

需要记住的一些额外要点：

  • 你可以通过点击角色来循环切换消息块的角色，当你在 Assistant 块中收到响应并想要编辑并将其作为 You 块发送回去时，这很有用。

命令概览

斜杠命令增强了助手的功能。在行首输入 / 开始查看可用命令列表：

  • /default：插入默认规则
  • /diagnostics：注入项目语言服务器报告的错误
  • /fetch：获取网页内容并插入
  • /file：插入单个文件或文件目录
  • /now：插入当前日期和时间
  • /prompt：向上下文添加自定义配置的提示词（参见规则库）
  • /symbols：插入当前标签页的活动符号
  • /tab：插入活动标签页或所有打开标签页的内容
  • /terminal：插入终端输出的指定行数
  • /selection：插入选中的文本

注意：请记住，命令仅在创建文本线程或插入命令时才会被执行，因此像 /now 这样的命令不会持续更新，/file 命令也不会保持其内容最新。

/default

更多关于 /default 的信息请参见规则：编辑默认规则部分。

用法：/default

/diagnostics

/diagnostics 命令将项目语言服务器报告的错误注入到上下文中。这对于获取项目中当前问题的概览很有用。

用法：/diagnostics [--include-warnings] [path]

  • --include-warnings：可选标志，用于在错误之外还包含警告。
  • path：可选路径，用于将诊断限制为特定文件或目录。

/file

/file 命令将单个文件或文件目录的内容插入到上下文中。这允许你在与助手的对话中引用项目的特定部分。

用法：/file <path>

你可以使用 glob 模式来匹配多个文件或目录。

示例：

  • /file src/index.js - 将 src/index.js 的内容插入到上下文中。
  • /file src/*.js - 将 src 目录中所有 .js 文件的内容插入。
  • /file src - 将 src 目录中所有文件的内容插入。

/now

/now 命令将当前日期和时间插入到上下文中。这对于让语言模型知道当前时间（以及其知识库的时效性）很有用。

用法：/now

/prompt

/prompt 命令将提示词库中的提示词插入到上下文中。它还可用于在提示词中嵌套提示词。

用法：/prompt <prompt_name>

相关：/default

/symbols

/symbols 命令将当前标签页的活动符号（函数、类等）插入到上下文中。这对于获取当前文件结构的概览很有用。

用法：/symbols

/tab

/tab 命令将活动标签页或所有打开标签页的内容插入到上下文中。这允许你引用当前正在处理的内容。

用法：/tab [tab_name|all]

  • tab_name：要插入的特定标签页的可选名称。
  • all：插入所有打开标签页的内容。

示例：

  • /tab - 插入活动标签页的内容。
  • /tab "index.js" - 插入名为 "index.js" 的标签页内容。
  • /tab all - 插入所有打开标签页的内容。

/terminal

/terminal 命令将终端输出的指定行数插入到上下文中。这对于引用最近的命令输出或日志很有用。

用法：/terminal [<number>]

  • <number>：可选参数，指定要插入的行数（默认为 50 行）。

/selection

/selection 命令将编辑器中选中的文本插入到上下文中。这对于引用代码的特定部分很有用。

这等效于 agent: quote selection 命令（cmd->|ctrl->）。

用法：/selection

规则库中的命令

命令可以在规则库（之前称为提示词库）的规则中使用，以插入动态内容或执行操作。 例如，如果你想要创建一个对模型了解日期很重要的规则，你可以使用 /now 命令来插入当前日期。

警告：规则中的斜杠命令仅在文本线程中使用时才有效。在非文本线程中使用不被支持。

注意：规则中的斜杠命令必须独占一行。

参见上方的命令列表了解更多关于命令的信息，以及有哪些斜杠命令可用。

示例

你是一名专业的 Rust 工程师。用户要求你审查他们的项目并回答一些问题。

以下是关于他们项目的一些信息：

/file Cargo.toml

在上面的示例中，/file 命令用于将 Cargo.toml 文件（或项目中存在的所有 Cargo.toml 文件）的内容插入到规则中。

嵌套规则

类似于向默认规则添加规则，你可以使用 /prompt 命令在其他规则中嵌套规则（目前仅在文本线程中支持）。

你可能想要嵌套规则来：

  • 即时创建模板
  • 将文档或引用等集合分解为更小的、可混合匹配的部分
  • 创建类似规则的变体（例如，Async Rust - Tokio vs. Async Rust - Async-std）

示例：

标题：Zed 风格的 Rust

## 关于 Zed

/prompt Zed: Zed（关于 Zed 是什么的规则）

## Rust - Zed 风格

/prompt Rust: Async - Async-std（Zed 不使用 tokio）
/prompt Rust: Zed 风格的 Crates（我们有一些独特的约定）
/prompt Rust - Workspace deps（倾向于重用 workspace 中的依赖）

上面括号中的文本是注释，不是规则的一部分。

注意：虽然从技术上讲你可以在规则内部嵌套自身，但我们强烈不建议这样做。请自行承担风险！

通过使用嵌套规则，你可以创建模块化和可重复使用的规则组件，这些组件可以以各种方式组合以适应不同的场景。

注意：当使用斜杠命令引入额外上下文时，注入的内容可以直接在文本线程中内联编辑——这里的编辑不会传播到保存的规则中。

扩展性

扩展可以提供额外的斜杠命令。

参见扩展：斜杠命令了解如何创建你自己的命令。

高级概念

规则模板

Zed 使用规则模板来支持内部助手功能，例如终端助手，或内联助手中使用的内容规则。

Zed 有以下内部规则模板：

  • content_prompt.hbs：用于在编辑器中生成内容。
  • terminal_assistant_prompt.hbs：用于终端助手功能。

目前还不确定我们是否会进一步扩展模板，使其可供用户创建。

覆盖模板

注意：除非你知道自己在做什么，否则不建议覆盖模板。如果操作不当，编辑模板会破坏你的助手。

Zed 允许你通过在 ~/.config/zed/prompt_overrides 目录中放置自定义 Handlebars (.hbs) 模板来覆盖各种助手功能使用的默认规则。

以下模板可以被覆盖：

  1. content_prompt.hbs：用于在编辑器中生成内容。

  2. terminal_assistant_prompt.hbs：用于终端助手功能。

注意：请确定你想要覆盖这些，因为你会错过我们内置功能的迭代更新。这主要应该在开发 Zed 时使用。

你可以自定义这些模板以更好地满足你的需求，同时保持 Zed 使用的核心结构和变量。Zed 会在磁盘上的提示词覆盖文件发生更改时自动重新加载。

查阅 Zed 的 assets/prompts 目录获取你可以使用的当前版本。

历史记录

在文本线程中提交第一条消息后，语言模型会为你的上下文生成一个名称，上下文会自动保存到你的文件系统中：

  • ~/.config/zed/conversations (macOS)
  • ~/.local/share/zed/conversations (Linux)
  • %LocalAppData%\Zed\conversations (Windows)

你可以通过点击 agent 面板左上角的历史按钮来访问和加载以前的上下文。
