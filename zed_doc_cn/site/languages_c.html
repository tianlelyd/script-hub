C Clangd: 强制检测为 C 格式化 编译命令 CMake 编译命令 调试 构建和调试二进制文件

C

Zed 原生支持 C 语言。

  • 语法高亮（Tree-sitter）: tree-sitter/tree-sitter-c
  • 语言服务器: clangd/clangd
  • 调试适配器: CodeLLDB（主要）、GDB（次要，不适用于 Apple 芯片）

Clangd: 强制检测为 C

Clangd 开箱即用地假设项目是混合的 C++/C 项目。如果你有一个纯 C 项目，你可能希望指示 clangd 使用 -xc 标志将所有文件视为 C。要做到这一点，请在项目根目录中创建一个 .clangd 文件，内容如下：

CompileFlags:
  Add: [-xc]

默认情况下，clang 和 gcc 会识别 *.C 和 *.H（大写扩展名）为 C++ 而不是 C，所以 Zed 也遵循这一约定。如果你在使用纯 C 项目（可能是带有传统大写路径的项目，如 FILENAME.C），你可以通过在设置中添加以下内容来覆盖此行为：

{
  "file_types": {
    "C": ["C", "H"]
  }
}

格式化

默认情况下，Zed 会使用 clangd 语言服务器来格式化 C 代码。Clangd 的格式化功能与 clang-format CLI 工具相同。要配置此功能，你可以添加一个 .clang-format 文件。例如：

---
BasedOnStyle: GNU
IndentWidth: 2
---

查看 Clang-Format 样式选项 获取完整的选项列表。

你可以通过 cmd-shift-i|ctrl-shift-i 或从命令面板使用 editor: format 操作，或者在 Zed 设置中添加 format_on_save 来触发格式化：

  "languages": {
    "C": {
      "format_on_save": "on",
      "tab_size": 2
    }
  }

编译命令

对于某些项目，Clangd 需要一个 compile_commands.json 文件来正确分析你的项目。此文件包含编译数据库，告诉 clangd 如何构建你的项目。

CMake 编译命令

使用 CMake 时，你可以通过在 CMakeLists.txt 中添加以下行来自动生成 compile_commands.json：

set(CMAKE_EXPORT_COMPILE_COMMANDS ON)

构建项目后，CMake 将在构建目录中生成 compile_commands.json 文件，clangd 将自动获取它。

调试

你可以使用 CodeLLDB 或 GDB 来调试本地二进制文件。（确保你的构建过程向 C 编译器传递 -g 参数，以便在生成的二进制文件中包含调试信息。）请参阅以下可以添加到 .zed/debug.json 的调试配置示例。

构建和调试二进制文件

[
  {
    "label": "调试本地二进制文件",
    "build": {
      "command": "make",
      "args": ["-j8"],
      "cwd": "$ZED_WORKTREE_ROOT"
    },
    "program": "$ZED_WORKTREE_ROOT/build/prog",
    "request": "launch",
    "adapter": "CodeLLDB"
  }
]
