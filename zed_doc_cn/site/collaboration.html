协作

只与你信任的人协作。由于分享项目会给他们访问你本地文件系统的权限，你不应该与不信任的人分享项目；他们可能会做一些恶意的事情。

未来，我们将采取更多措施来防止这种超出共享项目范围的访问，并增加对协作者权限的控制，但目前，请只与你信任的人协作。

添加协作者到通话

在开始协作之前，你需要将协作者添加到你的联系人中。操作步骤：

  1. 通过点击窗口右上角的 显示联系人菜单 按钮或运行 collab: toggle contacts menu (cmd-shift-c) 打开联系人菜单。
  2. 点击搜索框右侧的添加按钮。
  3. 使用 GitHub 用户名搜索你想要添加的联系人。注意：你试图添加为联系人的人必须是现有的 Zed 用户。

邀请协作者

你可以从联系人菜单中添加现有的 Zed 用户为联系人，通过点击窗口右上角的 显示联系人菜单 按钮或 collab: toggle contacts menu (cmd-shift-c) 打开菜单，然后点击搜索框右侧的 搜索新联系人 按钮。

当你邀请协作者加入一个不在通话中的项目时，他们将收到加入的通知，并且会创建一个新的通话。

邀请非 Zed 用户

如果你想要协作的人还没有注册 Zed，他们需要先 下载应用 并首次登录，然后你才能添加他们。身份验证与 GitHub 账户绑定，所以新用户需要通过 GitHub 认证才能登录 Zed。

语音聊天

加入通话时，如果你的操作系统允许，Zed 将自动与通话中的其他用户共享你的麦克风。这与你的项目无关。你可以通过 mute_on_join 设置在客户端禁用此功能。

项目协作

分享项目

当你邀请协作者加入你的项目时，一个新的通话开始。你的 Zed 窗口将在窗口的标题栏中显示通话参与者。

与你在同一项目中的协作者显示为彩色，并具有光标颜色。在其他项目中的协作者显示为灰色。可以访问当前项目的协作者在他们的头像下方会有自己的光标颜色。

我们旨在尽可能消除本地和远程项目之间的区别。协作者可以打开、编辑和保存文件，执行搜索，与语言服务器交互等。访客具有项目的只读视图，包括访问语言服务器信息。

未共享的项目

如果协作者当前在一个未共享的项目中，你将无法跳转到他们的项目或跟随他们，直到他们共享项目或返回到一个已共享的项目。

如果你在一个未共享的项目中，其他人将无法加入或查看其内容。

跟随协作者

要跟随协作者，请点击窗口右上方的他们的头像。你也可以使用 workspace: follow next collaborator (ctrl-alt-cmd-f) 在协作者之间循环切换。

当你加入一个项目时，你将立即开始跟随邀请你的协作者。

当你在一个跟随协作者的面板中时，你将：

  • 跟随他们的光标和滚动位置
  • 跟随他们在同一项目中的其他文件
  • 如果他们正在共享屏幕并离开项目，则在该面板中立即切换到查看他们的屏幕

如果你移动光标或在该面板中进行编辑，你将停止跟随。

要重新开始跟随，你可以点击协作者的头像，或通过按 workspace: follow next collaborator (ctrl-alt-cmd-f) 在不同参与者之间循环跟随。

跟随功能的工作原理

跟随功能仅限于特定的面板。当一个面板正在跟随协作者时，它会用他们的光标颜色进行轮廓显示。

这种面板特定的行为允许你在一个面板中跟随某人，同时在另一个面板中独立导航，这对于某些协作风格可能是一种有效的布局。

分享你的屏幕

通过点击窗口右上方的 共享屏幕 按钮与当前通话中的协作者共享你的屏幕。

如果协作者正在跟随你，而你开始查看 Zed 之外的窗口或未共享的项目，他们将看到你的屏幕。

当你进行屏幕共享时，协作者可以看到你的整个屏幕，所以要小心不要分享你不想分享的内容。完成后记得停止屏幕共享。

如果你正在共享屏幕，通话参与者可以通过打开右上角的联系人菜单并点击 屏幕 项目来为你的屏幕共享打开一个专用选项卡。

添加项目

你可以通过点击标题栏中项目名称旁边的 共享 按钮来将项目添加到通话中。

移除项目

你可以通过点击标题栏中项目名称旁边的 取消共享 按钮来从通话中移除项目。

当前在该项目中的协作者将从项目中断开连接，除非你再次共享它，否则他们将无法重新加入。

跟随协作者的终端

你可以通过让协作者共享他们的屏幕并跟随它来观看协作者在他们的终端中做什么。

在未来，我们计划允许你在共享项目中直接在终端中进行协作。

离开通话

你可以通过打开右上角的联系人菜单并点击 离开通话 按钮来离开通话。
