编辑预测

编辑预测是 Zed 通过 AI 预测您想要编写代码的原生机制。 每次按键都会向我们的 开源、开放数据集的 Zeta 模型 发送一个新请求，它会返回单行或多行建议，可以通过按 tab 快速接受。

配置 Zeta

Zed 的编辑预测最初通过标题栏上的横幅进行介绍。 点击它会带您到一个模态框，其中有一个按钮（“启用编辑预测”），可以将 zed 设置为您的 edit_prediction_provider。

但是，如果您没有遇到该横幅，Zed 的编辑预测是默认的编辑预测提供者，您应该在状态栏中立即看到它。

切换模式

Zed 的编辑预测提供两种不同的显示模式：

  1. eager（默认）：只要与语言服务器完成不冲突，就在内联显示预测
  2. subtle：仅在按住修饰键（默认为 alt）时内联显示预测

通过 mode 键在它们之间切换：

"edit_predictions": {
  "mode": "eager" | "subtle"
},

与其他 tab 操作的冲突

默认情况下，当 tab 通常执行不同操作时，Zed 需要修饰键来接受预测：

  1. 当语言服务器完成菜单可见时。
  2. 当光标不在正确的缩进级别时。

在这些情况下，使用 alt-tab 来接受预测。当语言服务器完成菜单打开时，首先按住 alt 会使其暂时消失，以便在缓冲区内预览预测。

在 Linux 上，alt-tab 通常被窗口管理器用于切换窗口，因此提供 alt-l 作为接受预测的默认绑定。tab 和 alt-tab 也可以使用，但默认不显示。

editor: accept partial edit prediction (ctrl-cmd-right|alt-right) 可以用于接受当前编辑预测直到下一个单词边界。

参见下面的 配置 GitHub Copilot 和 配置 Supermaven 部分以了解其他提供者的配置。这些提供者只支持在当前光标处插入文本，而 Zeta 模型提供多种预测，包括删除。

配置编辑预测快捷键

默认情况下，使用 tab 来接受编辑预测。您可以通过在键盘映射中插入以下内容来使用另一个快捷键：

{
  "context": "Editor && edit_prediction",
  "bindings": {
    // 这里我们也允许使用 `alt-enter` 来接受预测
    "alt-enter": "editor::AcceptEditPrediction"
  }
}

当出现 与 tab 键的冲突时，Zed 使用不同的上下文来接受快捷键（edit_prediction_conflict）。如果您想使用不同的快捷键，可以在您的键盘映射中插入以下内容：

{
  "context": "Editor && edit_prediction_conflict",
  "bindings": {
    "ctrl-enter": "editor::AcceptEditPrediction" // 修改快捷键的示例
  }
}

如果您的快捷键包含修饰键（上例中的 ctrl），它还将用于预览编辑预测并暂时隐藏语言服务器完成菜单。

您也可以将此操作绑定到没有修饰键的快捷键。在这种情况下，Zed 将使用默认修饰键（alt）来预览编辑预测。

{
  "context": "Editor && edit_prediction_conflict",
  "bindings": {
    // 这里我们绑定 tab 来接受，即使有语言服务器完成
    // 或光标不在正确的缩进级别
    "tab": "editor::AcceptEditPrediction"
  }
}

要在有语言服务器完成菜单时保持使用修饰键来接受预测，但允许 tab 无论光标位置如何都能接受预测，您可以使用 showing_completions 进一步指定上下文：

{
  "context": "Editor && edit_prediction_conflict && !showing_completions",
  "bindings": {
    // 这里我们不需要修饰键，除非有语言服务器完成
    "tab": "editor::AcceptEditPrediction"
  }
}

快捷键示例：始终使用 Alt-Tab

下面的快捷键示例使 alt-tab 始终被使用，而不是有时使用 tab。您可能希望这样做，以便只需一个快捷键来接受编辑预测，因为 tab 的行为因上下文而异。

  {
    "context": "Editor && edit_prediction",
    "bindings": {
      "alt-tab": "editor::AcceptEditPrediction"
    }
  },
  // 将 `tab` 绑定回其原始行为。
  {
    "context": "Editor",
    "bindings": {
      "tab": "editor::Tab"
    }
  },
  {
    "context": "Editor && showing_completions",
    "bindings": {
      "tab": "editor::ComposeCompletion"
    }
  },

如果在 settings.json 中设置了 "vim_mode": true，则需要在上述设置之后添加额外的绑定，以使 tab 返回其原始行为：

  {
    "context": "(VimControl && !menu) || vim_mode == replace || vim_mode == waiting",
    "bindings": {
      "tab": "vim::Tab"
    }
  },
  {
    "context": "vim_mode == literal",
    "bindings": {
      "tab": ["vim::Literal", ["tab", "\u0009"]]
    }
  },

快捷键示例：在 Linux 上显示 Tab 和 Alt-Tab

虽然在 Linux 上支持 tab 和 alt-tab，但显示的是 alt-l。如果您的窗口管理器没有保留 alt-tab，并且您希望使用 tab 和 alt-tab，请在 keymap.json 中包含这些绑定：

  {
    "context": "Editor && edit_prediction",
    "bindings": {
      "tab": "editor::AcceptEditPrediction",
      // 可选：这使默认的 `alt-l` 绑定不起作用。
      "alt-l": null
    }
  },
  {
    "context": "Editor && edit_prediction_conflict",
    "bindings": {
      "alt-tab": "editor::AcceptEditPrediction",
      // 可选：这使默认的 `alt-l` 绑定不起作用。
      "alt-l": null
    }
  },

缺少快捷键

Zed 在 Editor && edit_prediction 和 Editor && edit_prediction_conflict 上下文中都需要为 editor: accept edit prediction 操作至少设置一个快捷键（上面了解更多）。

如果您之前在全局上下文中将默认快捷键绑定到不同的操作，您将无法预览或接受编辑预测。例如：

[
  // 您的键盘映射
  {
    "bindings": {
      // 全局将 `alt-tab` 绑定到不同的操作
      "alt-tab": "menu::SelectNext"
    }
  }
]

要解决这个问题，您可以指定自己的快捷键来接受编辑预测：

[
  // ...
  {
    "context": "Editor && edit_prediction_conflict",
    "bindings": {
      "alt-l": "editor::AcceptEditPrediction"
    }
  }
]

如果您想使用默认快捷键，可以通过将您的快捷键移动到更具体的上下文或将其更改为其他内容来释放它。

禁用自动编辑预测

您可以在不同级别上禁用编辑预测显示，包括完全不开启。

或者，如果您将 Zed 设置为您的提供者，请考虑 使用精细模式。

在缓冲区上

为了不让预测在您输入时自动出现，请在 settings.json 中设置：

{
  "show_edit_predictions": false
}

这将隐藏有预测可用的所有指示，无论您处于 哪种显示模式（仅在您将 Zed 作为提供者时有效）。 但是，您仍然可以通过执行 editor: show edit prediction 或按 alt-tab|alt-l 来手动触发编辑预测。

针对特定语言

要在使用特定语言时不让预测在您输入时自动出现，请在 settings.json 中设置：

{
  "language": {
    "python": {
      "show_edit_predictions": false
    }
  }
}

在特定目录中

要针对特定目录或文件禁用编辑预测，请在 settings.json 中设置：

{
  "edit_predictions": {
    "disabled_globs": ["~/.config/zed/settings.json"]
  }
}

完全关闭

要在所有提供者中完全关闭编辑预测，请明确将设置设置为 none，如下所示：

"features": {
  "edit_prediction_provider": "none"
},

配置 GitHub Copilot

要使用 GitHub Copilot 作为您的提供者，请在 settings.json 中设置：

{
  "features": {
    "edit_prediction_provider": "copilot"
  }
}

您应该能够通过点击状态栏中的 Copilot 图标并按照设置说明登录到 GitHub Copilot。

使用 GitHub Copilot 企业版

如果您的组织使用 GitHub Copilot 企业版，您可以通过在 settings.json 中指定企业 URI 来配置 Zed 使用您的企业实例：

{
  "edit_predictions": {
    "copilot": {
      "enterprise_uri": "https://your.enterprise.domain"
    }
  }
}

将 "https://your.enterprise.domain" 替换为 GitHub 企业版管理员提供的 URL（例如，https://foo.ghe.com）。

设置后，Zed 将通过您的企业端点路由 Copilot 请求。当您通过点击状态栏中的 Copilot 图标登录时，您将被重定向到您配置的企业 URL 以完成身份验证。所有其他 Copilot 功能和用法保持不变。

Copilot 可以提供多个完成选项，可以使用以下操作进行导航：

  • editor: next edit prediction (alt-tab|alt-])：循环到下一个编辑预测
  • editor: previous edit prediction (alt-shift-tab|alt-[)：循环到上一个编辑预测

配置 Supermaven

要使用 Supermaven 作为您的提供者，请在 settings.json 中设置：

{
  "features": {
    "edit_prediction_provider": "supermaven"
  }
}

您应该能够通过点击状态栏中的 Supermaven 图标并按照设置说明登录到 Supermaven。
