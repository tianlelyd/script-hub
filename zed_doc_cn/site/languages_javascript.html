JavaScript

Zed 原生支持 JavaScript。

  • Tree-sitter: tree-sitter/tree-sitter-javascript
  • Language Server: typescript-language-server/typescript-language-server
  • Debug Adapter: vscode-js-debug

代码格式化

JavaScript 默认启用保存时格式化功能，使用 TypeScript 的内置代码格式化功能。 但许多 JavaScript 项目使用其他命令行代码格式化工具，如 Prettier。 你可以在设置中为 JavaScript 指定一个外部代码格式化工具来使用这些工具。 更多信息请参阅配置文档。

例如，如果你安装了 Prettier 并将其添加到 PATH 中，你可以通过将以下内容添加到你的 settings.json 来使用它格式化 JavaScript 文件：

{
  "languages": {
    "JavaScript": {
      "formatter": {
        "external": {
          "command": "prettier",
          "arguments": ["--stdin-filepath", "{buffer_path}"]
        }
      }
    }
  }
}

JSX

Zed 开箱即用地支持 JSX 语法高亮。

在 JSX 字符串中，tailwindcss-language-server 用于为 Tailwind CSS 类提供自动完成功能。

JSDoc

Zed 支持 JavaScript 和 TypeScript 注释中符合 JSDoc 语法的 JSDoc 语法。 Zed 使用 tree-sitter/tree-sitter-jsdoc 来解析和高亮 JSDoc。

ESLint

你可以配置 Zed 在格式化时运行 ESLint 代码操作来使用 eslint --fix 格式化代码：

{
  "languages": {
    "JavaScript": {
      "code_actions_on_format": {
        "source.fixAll.eslint": true
      }
    }
  }
}

你也可以在使用 fixAll 时只执行单个 ESLint 规则：

{
  "languages": {
    "JavaScript": {
      "code_actions_on_format": {
        "source.fixAll.eslint": true
      }
    }
  },
  "lsp": {
    "eslint": {
      "settings": {
        "codeActionOnSave": {
          "rules": ["import/order"]
        }
      }
    }
  }
}

注意： 你配置的其他格式化工具仍会在 ESLint 之后运行。 所以如果你的语言服务器或 Prettier 配置没有按照 ESLint 的规则格式化， 那么它们会覆盖 ESLint 修复的内容，导致你最终仍有错误。

如果你只想在保存时运行 ESLint，你可以将代码操作配置为格式化工具：

{
  "languages": {
    "JavaScript": {
      "formatter": {
        "code_actions": {
          "source.fixAll.eslint": true
        }
      }
    }
  }
}

配置 ESLint 的 nodePath：

你可以配置 ESLint 的 nodePath 设置：

{
  "lsp": {
    "eslint": {
      "settings": {
        "nodePath": ".yarn/sdks"
      }
    }
  }
}

配置 ESLint 的 problems：

你可以配置 ESLint 的 problems 设置。

例如，以下是如何设置 problems.shortenToSingleLine：

{
  "lsp": {
    "eslint": {
      "settings": {
        "problems": {
          "shortenToSingleLine": true
        }
      }
    }
  }
}

配置 ESLint 的 rulesCustomizations：

你可以配置 ESLint 的 rulesCustomizations 设置：

{
  "lsp": {
    "eslint": {
      "settings": {
        "rulesCustomizations": [
          // 将所有 eslint 错误/警告设置为显示为警告
          { "rule": "*", "severity": "warn" }
        ]
      }
    }
  }
}

配置 ESLint 的 workingDirectory：

你可以配置 ESLint 的 workingDirectory 设置：

{
  "lsp": {
    "eslint": {
      "settings": {
        "workingDirectory": {
          "mode": "auto"
        }
      }
    }
  }
}

调试

Zed 开箱即用地支持调试 JavaScript 代码。 以下内容可以在不编写额外配置的情况下进行调试：

  • 来自 package.json 的任务
  • 使用几种流行框架编写的测试（Jest、Mocha、Vitest、Jasmine）

运行 debugger: start (f4|f4) 可查看这些预定义调试任务的上下文列表。

与所有语言一样，.vscode/launch.json 中的配置也可用于在 Zed 中进行调试。

如果你的用例不在这些范围内，你可以通过向 .zed/debug.json 添加调试配置来完全控制。请参阅以下示例配置。

调试当前文件

[
  {
    "adapter": "JavaScript",
    "label": "调试 JS 文件",
    "type": "node",
    "request": "launch",
    "program": "$ZED_FILE",
    "skipFiles": ["<node_internals>/**"]
  }
]

这会隐式地使用 node 运行当前文件。

在 Chrome 中启动 Web 应用

[
  {
    "adapter": "JavaScript",
    "label": "在 Chrome 中调试应用",
    "type": "chrome",
    "request": "launch",
    "file": "$ZED_WORKTREE_ROOT/index.html",
    "webRoot": "$ZED_WORKTREE_ROOT",
    "console": "integratedTerminal",
    "skipFiles": ["<node_internals>/**"]
  }
]
