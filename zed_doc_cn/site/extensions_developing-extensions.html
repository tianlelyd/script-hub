开发扩展

扩展功能

扩展可以为 Zed 添加以下功能：

  • 语言
  • 调试器
  • 主题
  • 图标主题
  • 斜线命令
  • MCP 服务器

本地开发扩展

在开始为 Zed 开发扩展之前，请确保通过 rustup 安装 Rust。

必须通过 rustup 安装 Rust。如果您通过 homebrew 或其他方式安装了 Rust，安装开发扩展将不会正常工作。

在开发扩展时，您可以通过将其安装为开发扩展来在 Zed 中使用它，而无需发布。

从扩展页面，点击 Install Dev Extension 按钮（或使用 zed: install dev extension 操作）并选择包含您扩展的目录。

如果您需要排除故障，可以检查 Zed.log（zed: open log）以获取附加输出。对于调试输出，请关闭并使用命令行中的 zed --foreground 重新启动 Zed，这将显示更详细的 INFO 级别日志。

如果您已经安装了同名的已发布扩展，您的开发扩展将覆盖它。

安装后，Extensions 页面将显示上游扩展为 "被开发扩展覆盖"。

同名的预安装扩展必须在安装开发扩展之前卸载。更多信息请参阅 #31106。

Zed 扩展的目录结构

Zed 扩展是一个包含 extension.toml 的 Git 仓库。该文件必须包含有关扩展的一些基本信息：

id = "my-extension"
name = "我的扩展"
version = "0.0.1"
schema_version = 1
authors = ["Your Name <you@example.com>"]
description = "我的酷扩展"
repository = "https://github.com/your-name/my-zed-extension"

除此之外，还有几个其他可选的文件和目录可用于为 Zed 扩展添加功能。一个提供所有功能的扩展的示例目录结构如下：

my-extension/
  extension.toml
  Cargo.toml
  src/
    lib.rs
  languages/
    my-language/
      config.toml
      highlights.scm
  themes/
    my-theme.json

WebAssembly

扩展的程序部分用 Rust 编写并编译为 WebAssembly。要开发包含自定义代码的扩展，请包含一个如下所示的 Cargo.toml：

[package]
name = "my-extension"
version = "0.0.1"
edition = "2021"

[lib]
crate-type = ["cdylib"]

[dependencies]
zed_extension_api = "0.1.0"

使用 crates.io 上可用的最新版本的 zed_extension_api。确保它仍然与您要支持的 Zed 版本兼容。

在您的 Rust crate 的 src/lib.rs 文件中，您需要为您的扩展定义一个结构体并实现 Extension trait，同时使用 register_extension! 宏来注册您的扩展：

use zed_extension_api as zed;

struct MyExtension {
    // ... 状态
}

impl zed::Extension for MyExtension {
    // ...
}

zed::register_extension!(MyExtension);

stdout/stderr 直接转发到 Zed 进程。为了查看您扩展中的 println!/dbg! 输出，您可以在终端中使用 --foreground 标志启动 Zed。

Fork 和克隆仓库

  1. Fork 仓库

注意：如果您将 zed-industries/extensions 仓库 fork 到个人 GitHub 账户而不是 GitHub 组织，这将非常有帮助，因为这可以允许 Zed 工作人员向您的 PR 推送任何需要的更改以加快发布进程。

  1. 将仓库克隆到您的本地机器
# 在这里替换为您 fork 的 URL:
# git clone https://github.com/zed-industries/extensions
cd extensions
git submodule init
git submodule update

发布您的扩展

要发布扩展，请向 zed-industries/extensions 仓库开启 PR。

在您的 PR 中，请执行以下操作：

  1. 在 extensions/ 目录中将您的扩展添加为 Git 子模块
git submodule add https://github.com/your-username/foobar-zed.git extensions/foobar
git add extensions/foobar

所有扩展子模块必须使用 HTTPS URL 而不是 SSH URL（git@github.com）。

  1. 在顶级 extensions.toml 文件中添加一个包含您扩展的新条目：
[my-extension]
submodule = "extensions/my-extension"
version = "0.0.1"

如果您的扩展在子模块的子目录中，您可以使用 path 字段指向扩展所在的位置。

  1. 运行 pnpm sort-extensions 以确保 extensions.toml 和 .gitmodules 已排序

一旦您的 PR 被合并，扩展将被打包并发布到 Zed 扩展注册表。

扩展 ID 和名称不应包含 zed 或 Zed，因为它们都是 Zed 扩展。

更新扩展

要更新扩展，请向 zed-industries/extensions 仓库开启 PR。

在您的 PR 中请执行以下操作：

  1. 将扩展的子模块更新到新版本的提交。
  2. 在 extensions.toml 中更新扩展的 version 字段
    • 确保 version 与特定提交中 extension.toml 设置的版本匹配。

如果您希望自动化此进程，可以使用这个社区 GitHub Action。
