Agent 面板

Agent 面板允许您与多种大语言模型和编程 Agent 交互，它们可以帮助您完成各种任务，如生成代码、理解代码库以及其他一般性查询（如写邮件、文档等）。

要打开它，请在命令面板中使用 agent: new thread 操作，或点击状态栏中的 ✨（火花）图标。

快速开始

如果您是第一次使用 Agent 面板，需要配置至少一个大语言模型提供商或外部 Agent。 您可以通过以下方式配置：

  1. subscribing to our Pro plan, so you have access to our hosted models
  2. bringing your own API keys for your desired provider
  3. using an external agent like Gemini CLI or Claude Code

概览

配置好大语言模型提供商或外部 Agent 后，在消息编辑器中输入内容并按 Enter 键提交您的提示。 如果您需要更多的输入空间，可以使用 shift-alt-escape|shift-alt-escape 扩展消息编辑器。

您将开始看到响应流式传入，同时显示模型正在使用哪些工具来完成您的提示。 从这个时候开始，您可以使用下面概述的许多支持的功能。

请注意，对于外部 Agent（如 Gemini CLI 或 Claude Code），下面概述的一些功能目前尚未支持——例如，从历史记录中恢复线程、检查点、令牌使用量显示、模型选择等功能。我们希望在未来支持所有这些功能。

创建新线程

默认情况下，Agent 面板使用 Zed 的内置 Agent。

要更改这一点，请转到 Agent 面板右上角的加号按钮并选择其他选项。 您可以选择创建一个新的文本线程，或者如果您已连接了外部 Agent，也可以与它们创建新线程。

编辑消息

您发送给 AI 的任何消息都可以编辑。 您可以点击包含您消息的卡片，并使用调整后的提示和/或新的上下文内容重新提交。

检查点

每当 AI 执行编辑操作时，您应该在消息顶部看到一个“恢复检查点”按钮，允许您将代码库返回到该消息之前的状态。

即使您在编辑尝试进行到一半时中断了线程，检查点按钮也会出现，因为这可能是您发现 Agent 没有朝着正确的方向发展并想要返回的时候。

导航历史记录

要快速在最近打开的线程中导航，可以在面板编辑器获得焦点时使用 cmd-shift-j|ctrl-shift-j 快捷键，或者点击面板右上角的菜单图标按钮打开下拉菜单，显示最近的六个线程。

此菜单中的项目功能类似于选项卡，关闭它们不会删除线程；相反，它只是将它们从最近列表中删除。

要查看所有历史对话，请在同一菜单中选择 View All 选项或使用 cmd-shift-h|ctrl-shift-h 快捷键。

跟随 Agent

Zed 内置了协作功能。 这种方式也扩展到与 AI 的协作。 要跟随 Agent 阅读您的代码库并执行编辑，请点击面板左下角的"十字准星"图标按钮。

您也可以通过键盘操作实现：在提交消息时同时按下 cmd/ctrl 修饰键和 enter 键。

获取通知

如果您向 Agent 发送了提示，然后转到其他地方，将 Zed 放在后台，您可以通过以下方式获得响应完成的通知：

  • 在您屏幕右上角出现的视觉通知
  • 或者声音通知

您可以根据自己的喜好，同时使用两种通知方式或单独使用其中一种。

您可以通过使用 agent.notify_when_agent_waiting 和 agent.play_sound_when_agent_done 设置键来自定义它们的行为，包括完全关闭它们。

查看更改

一旦 Agent 对您的项目做出了更改，面板将显示哪些文件以及有多少文件被编辑过。

要查看具体哪些文件被编辑了，请展开消息编辑器上方显示的折叠栏，或点击 Review Changes 按钮（shift-ctrl-r|shift-ctrl-r），该按钮会打开一个显示所有更改的多缓冲区选项卡。

您可以拒绝或接受每个单独的更改块，或者 Agent 做出的整套更改。

编辑差异也会在各个缓冲区中显示。 因此，如果您的活动选项卡有 AI 做出的编辑，您将看到与多缓冲区中相同的接受/拒绝控件的差异。

添加上下文

尽管 Zed 的 Agent 在阅读您的代码库以自主获取相关文件、目录和其他上下文方面非常高效，但仍然鼓励手动添加上下文，作为加速和提高 AI 响应质量的方法。

要添加任何文件、目录、符号、之前的线程、规则文件甚至网页作为上下文，请在编辑器中输入 @ 来提及它们。

Agent 面板也支持粘贴图像作为上下文。

令牌使用量

Zed 在面板消息编辑器中配置文件选择器附近显示您当前活动线程消耗的令牌数量。 根据您添加的上下文片段数量，您的令牌消耗可能会迅速增长。

考虑到这一点，一旦您接近模型的上下文窗口，消息编辑器下方会出现一个横幅，建议开始一个新线程，并将当前线程总结并添加为上下文。 您也可以通过右上角的“Agent Options”菜单在任何时候对正在进行的线程执行此操作。

切换模型

在您配置了 LLM 提供商——无论是通过自定义 API 密钥还是通过Zed 的托管模型——之后，您可以通过点击消息编辑器上的模型选择器或使用 cmd-alt-/|ctrl-alt-/ 快捷键在它们之间切换。

使用工具

新的 Agent 面板支持工具调用，这使得 Agent 可以进行编辑。 Zed 带有多个内置工具，允许模型执行各种任务，如搜索代码库、编辑文件、运行命令等。

您也可以通过 MCP 服务器扩展可用工具集。

配置文件

配置文件作为对工具进行分组的方式。 Zed 提供三个内置配置文件，您可以创建任意数量的自定义配置文件。

Built-in Profiles

  • Write：具有工具的配置文件，允许 LLM 写入您的文件并运行终端命令。这个配置文件基本上开启了所有内置工具。
  • Ask：具有只读工具的配置文件。适合在不担心 Agent 做出更改的情况下询问有关代码库的问题。
  • Minimal：没有工具的配置文件。适合与 LLM 进行一般性对话，不需要代码库知识。

您可以通过点击配置文件选择器按钮 > Configure Profiles… > 您想要查看的配置文件来探索每个配置文件中启用的具体工具。

Custom Profiles

You can create a custom profile via the Configure Profiles… option in the profile selector. From here, you can choose to Add New Profile or fork an existing one with a custom name and your preferred set of tools.

You can also override built-in profiles. With a built-in profile selected, in the profile selector, navigate to Configure Tools, and select the tools you'd like.

Zed will store this profile in your settings using the same profile name as the default you overrode.

All custom profiles can be edited via the UI or by hand under the assistant.profiles key in your settings.json file.

Tool Approval

Zed's Agent Panel surfaces the agent.always_allow_tool_actions setting that, if turned to false, will require you to give permission to any editing attempt as well as tool calls coming from MCP servers.

You can change that by setting this key to true in either your settings.json or via the Agent Panel's settings view.

Model Support

Tool calling needs to be individually supported by each model and model provider. Therefore, despite the presence of tools, some models may not have the ability to pick them up yet in Zed. You should see a "No tools" label if you select a model that falls into this case.

We want to support all of them, though! We may prioritize which ones to focus on based on popularity and user feedback, so feel free to help and contribute to fast-track those that don't fit this bill.

All Zed's hosted models support tool calling out-of-the-box.

MCP Servers

Similarly to the built-in tools, some models may not support all tools included in a given MCP Server. Zed's UI will inform about this via a warning icon that appears close to the model selector.

文本线程

"Text Threads" present your conversation with the LLM in a different format—as raw text. With text threads, you have full control over the conversation data. You can remove and edit responses from the LLM, swap roles, and include more context earlier in the conversation.

For users who have been with us for some time, you'll notice that text threads are our original assistant panel—users love it for the control it offers. We do not plan to deprecate text threads, but it should be noted that if you want the AI to write to your code base autonomously, that's only available in the newer, and now default, "Threads".

错误和调试

In case of any error or strange LLM response behavior, the best way to help the Zed team debug is by reaching for the agent: open thread as markdown action and attaching that data as part of your issue on GitHub.

You can also open threads as Markdown by clicking on the file icon button, to the right of the thumbs down button, when focused on the panel's editor.

反馈

Every change we make to Zed's system prompt and tool set, needs to be backed by a thorough eval with good scores.

Every time the LLM performs a weird change or investigates a certain topic in your code base incorrectly, it's an indication that there's an improvement opportunity.

Note that rating responses will send your data related to that response to Zed's servers. See AI Improvement and Privacy and Security for more information about Zed's approach to AI improvement, privacy, and security. If you don't want data persisted on Zed's servers, don't rate. We will not collect data for improving our Agentic offering without you explicitly rating responses.

The best way you can help influence the next change to Zed's system prompt and tools is by rating the LLM's response via the thumbs up/down buttons at the end of every response. In case of a thumbs down, a new text area will show up where you can add more specifics about what happened.

You can provide feedback on the thread at any point after the agent responds, and multiple times within the same thread.
