Zed AI 改进

Agent 面板

选择加入

在使用 Agent 面板时，无论是通过 Zed 的托管 AI 服务，还是通过 API 密钥连接非 Zed AI 服务，Zed 都不会持久化存储用户内容或使用用户内容来评估和/或改进我们的 AI 功能，除非明确与 Zed 共享。每次共享都是选择性的，共享一次不会导致未来的内容或数据再次被共享。

请注意，对响应进行评分会将与该响应相关的数据发送到 Zed 的服务器。 如果您不希望数据被持久化存储在 Zed 服务器上，请不要进行评分。我们不会在您未明确对响应进行评分的情况下收集数据来改进我们的智能体产品。

当通过 Zed AI 使用上游服务时，我们要求服务提供商保证您的用户内容不会用于训练模型。例如，在助手中通过 Zed AI 使用 Anthropic Claude 3.5 遵循 Anthropic 商业条款，其中包括以下内容：

"对于付费服务产生的客户内容，Anthropic 不得使用其训练模型。"

我们还与 Anthropic 签署了零数据保留协议。

当您直接将 Zed 与非 Zed AI 服务（例如，通过 API 密钥）连接时，Zed 无法控制该服务提供商如何使用您的数据。 您应该参考与各服务提供商的协议，以了解适用的条款和条件。

我们收集的数据

对于您明确与我们共享的提示，Zed 可能会存储这些提示的副本以及关于 Agent 面板具体使用情况的其他数据。

此数据包括：

  • 给予 Agent 的提示
  • 您包含的任何评论
  • 关于智能体线程的产品遥测
  • 关于您的 Zed 安装的元数据

数据处理

收集的数据存储在 Snowflake 中，这是一个私有数据库，我们在其中跟踪其他指标。我们定期审查这些数据，以改进我们的整体智能体方法，并通过系统提示、工具使用等方式优化产品。我们确保包含的任何数据都被匿名化，并且不包含敏感信息（访问令牌、用户 ID、邮箱地址等）。

编辑预测

默认情况下，在使用 Zed 编辑预测时，Zed 不会持久化存储用户内容或使用用户内容来训练模型。

选择加入

正在处理开源许可项目的用户可以选择性地选择提供模型改进反馈。这种选择是基于项目的。如果您处理多个开源项目并希望提供模型改进反馈，您将必须为每个单独的项目选择加入。

在处理其他您未选择加入的项目时，Zed 不会持久化存储用户内容或使用用户内容来训练模型。

您可以在 license_detection.rs 中准确了解 Zed 如何检测开源许可证。

排除项

Zed 会有意地完全排除某些文件的预测编辑，即使您已选择加入模型改进反馈。

您可以通过在命令面板中打开 zed: open default settings 来查看此排除列表：

{
  "edit_predictions": {
    // 表示应禁用编辑预测的文件的 glob 列表。
    // 已经包含了一个合理的默认 glob 列表。
    // 对此列表的任何添加都将与默认列表合并。
    "disabled_globs": [
      "**/.env*",
      "**/*.pem",
      "**/*.key",
      "**/*.cert",
      "**/*.crt",
      "**/secrets.yml"
    ]
  }
}

用户可以通过在其 Zed settings.json 中将其添加到 edit_predictions.disabled_globs 来明确排除其他路径和/或文件扩展名：

{
  "edit_predictions": {
    "disabled_globs": ["secret_dir/*", "**/*.log"]
  }
}

我们收集的数据

对于您已选择加入的开源项目，Zed 可能会存储对 Zed AI 预测服务的请求和响应副本。

此数据包括：

  • 编辑预测
  • 光标周围的一部分缓冲区内容
  • 一些最近的编辑
  • 当前缓冲区大纲
  • 来自语言服务器的诊断（错误、警告等）

数据处理

收集的数据存储在 Snowflake 中，这是一个私有数据库，我们在其中跟踪其他指标。我们定期审查这些数据，以选择训练样本纳入我们的模型训练数据集。我们确保包含的任何数据都被匿名化，并且不包含敏感信息（访问令牌、用户 ID、邮箱地址等）。此训练数据集可在 huggingface.co/datasets/zed-industries/zeta 公开获取。

模型输出

然后，我们使用此训练数据集来微调 Qwen2.5-Coder-7B，并在 huggingface.co/zed-industries/zeta 提供结果模型。

适用条款

有关更多信息，请参阅 Zed 服务条款。
