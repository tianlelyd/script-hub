Java

Zed 中的 Java 语言支持由以下组件提供：

  • Zed Java: zed-extensions/java
  • Tree-sitter：tree-sitter/tree-sitter-java
  • 语言服务器：eclipse-jdtls/eclipse.jdt.ls

安装 OpenJDK

您需要安装 Java 运行时（OpenJDK）。

  • MacOS: brew install openjdk
  • Ubuntu: sudo add-apt-repository ppa:openjdk-23 && sudo apt-get install openjdk-23
  • Windows: choco install openjdk
  • Arch Linux: sudo pacman -S jre-openjdk-headless

或者手动下载并安装 OpenJDK 23。

扩展安装

您可以通过打开 zed: extensions（cmd-shift-x|ctrl-shift-x）并搜索 java 来安装扩展。

设置 / 初始化选项

扩展将自动下载语言服务器，如果您希望自己管理，请参见下面的：手动安装 JDTLS。

有关可用的 initialization_options，请参阅 Eclipse.jdt.ls Wiki 的 Initialize Request 部分。

您可以通过启动 zed: open settings（cmd-,|ctrl-,）或在项目中使用 .zed/setting.json 文件来将这些自定义设置添加到您的 Zed 设置中。

Zed Java 设置

{
  "lsp": {
    "jdtls": {
      "initialization_options": {}
    }
  }
}

示例配置

JDTLS 二进制文件

默认情况下，zed 将在您的 PATH 中查找 jdtls 二进制文件，如果您希望指定一个明确的二进制文件，可以通过设置来完成：

  "lsp": {
    "jdtls": {
      "binary": {
        "path": "/path/to/java/bin/jdtls",
        // "arguments": [],
        // "env": {},
        "ignore_system_version": true
      }
    }
  }

手动安装 JDTLS

如果您愿意，可以自己安装 JDTLS，并配置扩展使用该安装。

  • MacOS: brew install jdtls
  • Arch: 从 AUR 安装 jdtls

或者手动下载安装：

  • JDTLS 里程碑构建 （每两周更新）
  • JDTLS 快照构建 （频繁更新）
