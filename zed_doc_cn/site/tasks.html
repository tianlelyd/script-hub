任务

Zed 支持使用集成终端生成（和重新运行）命令来输出结果的方法。这些命令可以读取 Zed 状态的一个有限子集（例如当前正在编辑的文件路径或选中的文本）。

[
  {
    "label": "示例任务",
    "command": "for i in {1..5}; do echo \"Hello $i/5\"; sleep 1; done",
    //"args": [],
    // 命令的环境变量覆盖，将被附加到设置中终端的环境中。
    "env": { "foo": "bar" },
    // 生成命令的当前工作目录，默认为当前项目根目录。
    //"cwd": "/path/to/working/directory",
    // 是否使用新的终端选项卡或重复使用现有选项卡来生成进程，默认为 `false`。
    "use_new_terminal": false,
    // 是否允许运行同一任务的多个实例，或者等待现有实例完成，默认为 `false`。
    "allow_concurrent_runs": false,
    // 命令启动后，对终端面板和选项卡执行的操作：
    // * `always` — 总是显示任务的面板，并聚焦其中的相应选项卡（默认）
    // * `no_focus` — 总是显示任务的面板，在其中添加任务的选项卡，但不聚焦
    // * `never` — 不改变聚焦，但仍然在其面板中添加/重复使用任务的选项卡
    "reveal": "always",
    // 命令完成后，对终端面板和选项卡执行的操作：
    // * `never` — 命令完成时不做任何事（默认）
    // * `always` — 总是隐藏终端选项卡，如果是最后一个选项卡也隐藏面板
    // * `on_success` — 仅在任务成功时隐藏终端选项卡，否则与 `always` 的行为类似
    "hide": "never",
    // 在终端内运行任务时使用哪个 shell。
    // 可以取 3 个值：
    // 1. （默认）使用系统在 /etc/passwd 中的默认终端配置
    //      "shell": "system"
    // 2. 一个程序：
    //      "shell": {
    //        "program": "sh"
    //      }
    // 3. 带参数的程序：
    //     "shell": {
    //         "with_arguments": {
    //           "program": "/bin/bash",
    //           "args": ["--login"]
    //         }
    //     }
    "shell": "system",
    // 是否在生成任务的输出中显示任务行，默认为 `true`。
    "show_summary": true,
    // 是否在生成任务的输出中显示命令行，默认为 `true`。
    "show_output": true
    // 表示内联可运行指示器的标签，或同时生成多个任务。
    // "tags": []
  }
]

有两个操作驱动使用任务的工作流程：task: spawn 和 task: rerun。 task: spawn 打开一个模态框，显示当前文件中的所有可用任务。 task: rerun 重新运行最近生成的任务。您也可以从任务模态框中重新运行任务。

默认情况下，重新运行任务会重复使用同一个终端（由于 "use_new_terminal": false 默认设置），但在启动之前等待上一个任务完成（由于 "allow_concurrent_runs": false 默认设置）。

保持 "use_new_terminal": false 并设置 "allow_concurrent_runs": true 以允许在重新运行时取消上一个任务。

任务模板

任务可以定义在：

  • 在全局 tasks.json 文件中；这些任务在您处理的所有 Zed 项目中都可用。该文件通常位于 ~/.config/zed/tasks.json。您可以使用 zed: open tasks 操作来编辑它们。
  • 在工作树特定（本地）的 .zed/tasks.json 文件中；这些任务仅在处理包含该工作树的项目时可用。您可以使用 zed: open project tasks 操作来编辑工作树特定的任务。
  • 通过 一次性任务 动态创建。这些任务是项目特定的，不会在会话之间持久化。
  • 通过语言扩展。

变量

Zed 任务的行为就像您的 shell；这也意味着您可以通过类似 sh 的 $VAR_NAME 语法来引用环境变量。为了您的方便，设置了一些额外的环境变量。 这些变量允许您从当前编辑器中提取信息并在任务中使用。以下变量可用：

  • ZED_COLUMN：当前行列
  • ZED_ROW：当前行号
  • ZED_FILE：当前打开文件的绝对路径（例如 /Users/my-user/path/to/project/src/main.rs）
  • ZED_FILENAME：当前打开文件的文件名（例如 main.rs）
  • ZED_DIRNAME：当前打开文件去除文件名后的绝对路径（例如 /Users/my-user/path/to/project/src）
  • ZED_RELATIVE_FILE：当前打开文件相对于 ZED_WORKTREE_ROOT 的路径（例如 src/main.rs）
  • ZED_RELATIVE_DIR：当前打开文件目录相对于 ZED_WORKTREE_ROOT 的路径（例如 src）
  • ZED_STEM：当前打开文件的主名（不带扩展名的文件名）（例如 main）
  • ZED_SYMBOL：当前选中的符号；应该与符号面包屑中显示的最后一个符号匹配（例如 mod tests > fn test_task_contexts）
  • ZED_SELECTED_TEXT：当前选中的文本
  • ZED_WORKTREE_ROOT：当前工作树根目录的绝对路径（例如 /Users/my-user/path/to/project）
  • ZED_CUSTOM_RUST_PACKAGE：（Rust 特定）$ZED_FILE 源文件的父包名称。

要在任务中使用变量，请在其前面加上美元符号（$）：

{
  "label": "输出当前文件路径",
  "command": "echo $ZED_FILE"
}

您也可以使用详细语法，允许在给定变量不可用时指定默认值：${ZED_FILE:default_value}

这些环境变量也可以在任务的 cwd、args 和 label 字段中使用。

变量引用

在处理包含空格或其他特殊字符的路径时，请确保变量被正确转义。

例如，不要使用这种方式（如果路径中有空格将会失败）：

{
  "label": "查看当前文件状态",
  "command": "stat $ZED_FILE"
}

请使用以下方式：

{
  "label": "查看当前文件状态",
  "command": "stat",
  "args": ["$ZED_FILE"]
}

或者显式地包含转义引号，如下所示：

{
  "label": "查看当前文件状态",
  "command": "stat \"$ZED_FILE\""
}

一次性任务

通过 task: spawn 打开的同一个任务模态框支持任意的类 bash 命令执行：在模态框文本字段中输入命令，并使用 opt-enter 来生成它。

任务模态框在会话期间保持这些临时命令，如果它们是最后一次生成的任务，task: rerun 也会重新运行这些任务。

您也可以调整模态框中当前选中的任务（tab 是默认的键绑定）。这样做将将其命令放入提示符中，然后可以编辑并作为一次性任务生成。

临时任务

您可以在通过模态框生成任务时使用 cmd 修饰符；以这种方式生成的任务不会增加其使用计数（因此，它们不会被 task: rerun 重新生成，并且在任务模态框中也不会有高排名）。 临时任务的预期用途是在持续使用 task: rerun 的过程中保持在流程中。

任务的自定义快捷键

您可以通过向 task::Spawn 添加额外参数来为任务定义自己的快捷键。如果您想将上述的 echo current file's path 任务绑定到 alt-g，您将在 keymap.json 文件中添加以下代码片段：

{
  "context": "Workspace",
  "bindings": {
    "alt-g": ["task::Spawn", { "task_name": "输出当前文件路径" }]
  }
}

请注意，这些任务也可以指定“target”来控制生成的任务应该在哪里显示。 这对于启动您想在中央区域使用的终端应用程序可能很有用：

// In tasks.json
{
  "label": "启动 lazygit",
  "command": "lazygit -p $ZED_WORKTREE_ROOT"
}

// In keymap.json
{
  "context": "Workspace",
  "bindings": {
    "alt-g": [
      "task::Spawn",
      { "task_name": "启动 lazygit", "reveal_target": "center" }
    ]
  }
}

将可运行标签绑定到任务模板

Zed 支持通过工作区本地和全局 tasks.json 文件覆盖内联可运行指示器的默认操作，具有以下优先级层次结构：

  1. 工作区 tasks.json
  2. 全局 tasks.json
  3. 语言提供的标签绑定（默认）。

要标记一个任务，请在任务模板的 tags 字段中添加可运行标签名称：

{
  "label": "输出当前文件路径",
  "command": "echo $ZED_FILE",
  "tags": ["rust-test"]
}

通过这样做，您可以改变在可运行指示器中显示的任务。

运行绑定到可运行对象的任务的快捷键

当您有一个绑定到可运行对象的任务定义时，您可以使用 代码操作 快速运行它，您可以通过 editor: Toggle Code Actions 命令或 cmd-./ctrl-. 快捷键来触发。您的任务将是下拉列表中的第一个。如果此行没有其他代码操作，任务将立即运行。
