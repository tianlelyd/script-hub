模型上下文协议

Zed 使用 模型上下文协议 与上下文服务器进行交互。

模型上下文协议 (MCP) 是一个开放的协议，它允许 LLM 应用程序与外部数据源和工具无缝集成。无论您是在构建 AI 驱动的 IDE、增强聊天界面，还是创建自定义 AI 工作流，MCP 都提供了一种标准化的方式来将 LLM 与其所需的上下文连接起来。

查看 Anthropic 新闻文章 和 Zed 博客文章 以了解 MCP 的概述。

安装 MCP 服务器

作为扩展

在 Zed 中使用 MCP 服务器的方式之一是将它们作为扩展暴露。 要学习如何创建自己的扩展，请查看 MCP 服务器扩展 页面以获取更多详细信息。

感谢我们的优秀社区，许多 MCP 服务器已经被添加为扩展。 您可以通过以下任一路径查看可用的扩展：

  1. Zed 网站
  2. 在应用程序中，打开命令面板并运行 zed: extensions 操作
  3. 在应用程序中，转到 Agent 面板的右上角菜单，并查找 "查看服务器扩展" 菜单项

无论如何，以下是一些可用的扩展：

  • Context7
  • GitHub
  • Puppeteer
  • Gem
  • Brave Search
  • Prisma
  • Framelink Figma
  • Linear
  • Resend

作为自定义服务器

创建扩展并不是在 Zed 中使用 MCP 服务器的唯一方式。 您可以通过将它们的命令直接添加到您的 settings.json 来连接它们，如下所示：

{
  "context_servers": {
    "your-mcp-server": {
      "source": "custom",
      "command": "some-command",
      "args": ["arg-1", "arg-2"],
      "env": {}
    }
  }
}

另外，您也可以通过访问 Agent 面板的设置视图（也可通过 agent: open settings 操作访问）来添加自定义服务器。 在那里，您可以通过单击 "添加自定义服务器" 按钮时出现的模态框来添加它。

使用 MCP 服务器

安装检查

无论您是将 MCP 服务器作为扩展使用还是直接添加它们，大多数服务器都需要在设置过程中进行一些配置。

对于扩展情况，Zed 会显示一个模态框，显示正确设置特定服务器所需的内容。 例如，GitHub MCP 扩展需要您添加一个 个人访问令牌。

对于自定义服务器情况，请确保查看提供商文档，以确定需要在 JSON 中添加什么类型的命令、参数和环境变量。

要检查您的 MCP 服务器是否正确安装，请转到 Agent 面板的设置视图并观察其名称旁边的指示灯。 如果它们运行正常，指示灯将为绿色，其工具提示将显示 "服务器正在运行"。 如果不是，其他颜色和工具提示信息将指示发生了什么情况。

在 Agent 面板中使用

安装完成后，您可以返回 Agent 面板并开始提示。 按名称提及您的 MCP 服务器有助于 agent 识别它。

如果您希望确保使用特定的服务器，您可以通过关闭内置工具（所有工具或可能导致冲突的工具）并仅打开来自 MCP 服务器的工具来创建 自定义配置文件。

作为一个例子，Dagger 团队建议对他们的 Container Use MCP 服务器这样做：

"agent": {
  "profiles": {
    "container-use": {
      "name": "Container Use",
      "tools": {
        "fetch": true,
        "thinking": true,
        "copy_path": false,
        "find_path": false,
        "delete_path": false,
        "create_directory": false,
        "list_directory": false,
        "diagnostics": false,
        "read_file": false,
        "open": false,
        "move_path": false,
        "grep": false,
        "edit_file": false,
        "terminal": false
      },
      "enable_all_context_servers": false,
      "context_servers": {
        "container-use": {
          "tools": {
            "environment_create": true,
            "environment_add_service": true,
            "environment_update": true,
            "environment_run_cmd": true,
            "environment_open": true,
            "environment_file_write": true,
            "environment_file_read": true,
            "environment_file_list": true,
            "environment_file_delete": true,
            "environment_checkpoint": true
          }
        }
      }
    }
  }
}

工具批准

Zed 的 Agent 面板包含 agent.always_allow_tool_actions 设置，如果设置为 false，将要求您为任何编辑尝试以及来自 MCP 服务器的工具调用给予许可。

您可以通过在 settings.json 中或通过 Agent 面板的设置视图将此键设置为 true 来更改此设置。
