使用规则

规则本质上是在与 Agent 的每次交互开始时插入的提示。 目前，Zed 支持在目录根目录下的 .rules 文件和规则库，这允许您存储多个规则以供按需使用。

.rules 文件

Zed 支持在工作树的顶级包含 .rules 文件，它们作为项目级指令，包含在您与 Agent 面板的所有交互中。 为了与其他代理兼容，也支持此文件的其他名称，但请注意将使用此列表中第一个匹配的文件：

  • .rules
  • .cursorrules
  • .windsurfrules
  • .clinerules
  • .github/copilot-instructions.md
  • AGENT.md
  • AGENTS.md
  • CLAUDE.md
  • GEMINI.md

规则库

规则库是用于编写和管理规则的界面。与 Zed 中的其他文本驱动 UI 一样，它是一个功能完整的编辑器，具有语法高亮、键盘快捷键等功能。

您可以直接在规则编辑器中使用内联助手，允许您自动化和重写规则。

打开规则库

  1. 打开 Agent 面板。
  2. 点击右上角的 Agent 菜单（...）。
  3. 从下拉菜单中选择 规则...。

您也可以在 Agent 面板中使用 agent: open rules library 命令。

管理规则

选择规则文件后，您可以直接在内置编辑器中编辑它。其标题也可以从编辑器标题栏中更改。

规则可以使用规则编辑器中的按钮进行复制、删除或添加到默认规则中。

创建规则

要创建规则文件，只需打开规则库并点击 + 按钮。规则文件存储在本地，可以随时从库中访问。

拥有一系列专门针对提示工程定制的规则文件也可以帮助您编写一致且有效的规则。

以下是编写更好规则的几个有用资源：

  • Anthropic: Prompt Engineering
  • OpenAI: Prompt Engineering

编辑默认规则

Zed 允许您自定义与 LLM 交互时使用的默认规则。 更准确地说，它使用一系列组合形成默认规则的规则。

默认规则会自动包含在每个新线程的上下文中。 您还可以使用 @rule 命令手动添加其他规则（未标记为默认的规则）作为上下文。

从提示库迁移

以前，规则库被称为"提示库"。 新的规则系统替代了提示库，除了以下概述的少数特定情况。

规则中的斜杠命令

以前，可以在自定义提示（现在的规则）中使用斜杠命令（现在的@提及）。 目前不支持在规则文件中使用@提及，但是，当与文本线程一起使用时，规则文件支持斜杠命令。 有关更多信息，请参阅在规则中使用斜杠命令的文档。

提示模板

Zed 保持与其原始模板系统的向后兼容性，该系统允许您自定义整个应用程序中使用的提示，包括内联助手。 虽然规则库现在是管理提示的主要方式，但您仍然可以使用这些遗留模板来覆盖默认提示。 有关更多详细信息，请参阅文本线程下的规则模板部分。
