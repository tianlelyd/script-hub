TypeScript

Zed 原生支持 TypeScript 和 TSX。

  • 语法树解析器: tree-sitter/tree-sitter-typescript
  • 语言服务器: yioneko/vtsls
  • 备用语言服务器: typescript-language-server/typescript-language-server
  • 调试适配器: vscode-js-debug

语言服务器

默认情况下，Zed 对 TypeScript、TSX 和 JavaScript 文件使用 vtsls。 您可以在设置文件中为每种语言配置使用 typescript-language-server：

{
  "languages": {
    "TypeScript": {
      "language_servers": ["typescript-language-server", "!vtsls", "..."]
    },
    "TSX": {
      "language_servers": ["typescript-language-server", "!vtsls", "..."]
    },
    "JavaScript": {
      "language_servers": ["typescript-language-server", "!vtsls", "..."]
    }
  }
}

默认情况下，TypeScript 文件也会使用 Prettier。要禁用此功能：

{
  "languages": {
    "TypeScript": {
      "prettier": { "allowed": false }
    }
    //...
  }
}

大型项目

vtsls 在非常大的项目中可能会内存不足。我们将默认限制设置为 8092（8 GiB），而不是默认的 3072，但这对您来说可能还不够：

{
  "lsp": {
    "vtsls": {
      "settings": {
        // 针对 TypeScript：
        "typescript": { "tsserver": { "maxTsServerMemory": 16184 } },
        // 针对 JavaScript：
        "javascript": { "tsserver": { "maxTsServerMemory": 16184 } }
      }
    }
  }
}

内联提示

Zed 设置以下初始化选项以使语言服务器发送回内联提示（即当 Zed 在设置中启用内联提示时）。

当使用 typescript-language-server 时，您可以在 Zed settings.json 中覆盖这些设置：

{
  "lsp": {
    "typescript-language-server": {
      "initialization_options": {
        "preferences": {
          "includeInlayParameterNameHints": "all",
          "includeInlayParameterNameHintsWhenArgumentMatchesName": true,
          "includeInlayFunctionParameterTypeHints": true,
          "includeInlayVariableTypeHints": true,
          "includeInlayVariableTypeHintsWhenTypeMatchesName": true,
          "includeInlayPropertyDeclarationTypeHints": true,
          "includeInlayFunctionLikeReturnTypeHints": true,
          "includeInlayEnumMemberValueHints": true
        }
      }
    }
  }
}

更多信息请参见 typescript-language-server 内联提示文档。

当使用 vtsls 时：

{
  "lsp": {
    "vtsls": {
      "settings": {
        // 针对 JavaScript：
        "javascript": {
          "inlayHints": {
            "parameterNames": {
              "enabled": "all",
              "suppressWhenArgumentMatchesName": false
            },
            "parameterTypes": {
              "enabled": true
            },
            "variableTypes": {
              "enabled": true,
              "suppressWhenTypeMatchesName": true
            },
            "propertyDeclarationTypes": {
              "enabled": true
            },
            "functionLikeReturnTypes": {
              "enabled": true
            },
            "enumMemberValues": {
              "enabled": true
            }
          }
        },
        // 针对 TypeScript：
        "typescript": {
          "inlayHints": {
            "parameterNames": {
              "enabled": "all",
              "suppressWhenArgumentMatchesName": false
            },
            "parameterTypes": {
              "enabled": true
            },
            "variableTypes": {
              "enabled": true,
              "suppressWhenTypeMatchesName": true
            },
            "propertyDeclarationTypes": {
              "enabled": true
            },
            "functionLikeReturnTypes": {
              "enabled": true
            },
            "enumMemberValues": {
              "enabled": true
            }
          }
        }
      }
    }
  }
}

调试

Zed 开箱即用支持 TypeScript 代码调试。 以下内容可以在不编写额外配置的情况下进行调试：

  • 来自 package.json 的任务
  • 使用几种流行框架编写的测试（Jest、Mocha、Vitest、Jasmine）

运行 debugger: start（f4|f4）可查看这些预定义调试任务的上下文列表。

与所有语言一样，.vscode/launch.json 中的配置也可用于在 Zed 中进行调试。

如果您的用例不在其中任何一种情况内，您可以通过向 .zed/debug.json 添加调试配置来完全控制。请参见下面的示例配置。

附加调试器到在 Web 浏览器中运行的服务器（npx serve）

对于外部运行的 Web 服务器（例如，使用 npx serve 或 npx live-server），可以附加到它并用浏览器打开。

[
  {
    "label": "Launch Chrome (TypeScript)",
    "adapter": "JavaScript",
    "type": "chrome",
    "request": "launch",
    "url": "http://localhost:5500",
    "program": "$ZED_FILE",
    "webRoot": "${ZED_WORKTREE_ROOT}",
    "build": {
      "command": "npx",
      "args": ["tsc"]
    },
    "skipFiles": ["<node_internals>/**"]
  }
]
