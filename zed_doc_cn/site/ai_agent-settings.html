Agent 设置

了解在 Zed 的 Agent 面板中可以自定义的所有设置。

模型设置

默认模型

如果你正在使用 Zed 的托管 LLM 服务，它会将 claude-sonnet-4 设置为默认模型。 但如果你没有订阅该服务或者只是想更改它，你可以通过 Agent 面板右下角的模型下拉菜单或在设置中手动编辑 default_model 对象来完成：

{
  "agent": {
    "default_model": {
      "provider": "zed.dev",
      "model": "gpt-4o"
    }
  }
}

功能特定模型

为 Zed 中以下 AI 功能指定独特和特定的模型：

  • 线程摘要模型：用于生成线程摘要
  • 内联助手模型：用于内联助手功能
  • 提交信息模型：用于生成 Git 提交信息
{
  "agent": {
    "default_model": {
      "provider": "zed.dev",
      "model": "claude-sonnet-4"
    },
    "inline_assistant_model": {
      "provider": "anthropic",
      "model": "claude-3-5-sonnet"
    },
    "commit_message_model": {
      "provider": "openai",
      "model": "gpt-4o-mini"
    },
    "thread_summary_model": {
      "provider": "google",
      "model": "gemini-2.0-flash"
    }
  }
}

如果没有为其中任何一个功能设置自定义模型，它们会自动回退到使用默认模型。

内联助手的替代模型

内联助手功能特别具有使用不同模型并行执行多个生成的能力。 这可以通过为其分配多个模型来实现，进一步扩展上面显示的配置。

配置后，内联助手 UI 将显示控件来循环查看每个模型生成的输出。

你在这里指定的模型总是除了你的默认模型之外使用的。

例如，以下配置将为每次助手请求生成两个输出。 一个使用 Claude Sonnet 4（默认模型），一个使用 GPT-4o。

{
  "agent": {
    "default_model": {
      "provider": "zed.dev",
      "model": "claude-sonnet-4"
    },
    "inline_alternatives": [
      {
        "provider": "zed.dev",
        "model": "gpt-4o"
      }
    ]
  }
}

模型温度

为提供商和/或模型指定自定义温度：

"model_parameters": [
  // 为所有 OpenAI 模型请求设置参数：
  {
    "provider": "openai",
    "temperature": 0.5
  },
  // 为所有请求设置通用参数：
  {
    "temperature": 0
  },
  // 为特定提供商和模型设置参数：
  {
    "provider": "zed.dev",
    "model": "claude-sonnet-4",
    "temperature": 1.0
  }
],

Agent 面板设置

请注意，其中一些设置也可以在 Agent 面板的设置 UI 中找到，你可以通过 agent: open settings 操作或面板右上角的下拉菜单访问。

默认视图

使用 default_view 设置来更改 Agent 面板的默认视图。 你可以在 thread（默认）和 text_thread 之间选择：

{
  "agent": {
    "default_view": "text_thread"
  }
}

自动运行命令

控制是否允许 agent 在不询问你许可的情况下运行命令。 默认值是 false。

{
  "agent": {
    "always_allow_tool_actions": true
  }
}

此设置可通过 Agent 面板的设置 UI 获得。

单文件审查

控制是否希望在 agent 完成编辑后在单个缓冲区中看到审查操作（接受和拒绝）。 默认值是 false。

{
  "agent": {
    "single_file_review": true
  }
}

当设置为 false 时，这些控件仅在多缓冲区审查标签中可用。

此设置可通过 Agent 面板的设置 UI 获得。

声音通知

控制当 agent 完成生成更改或需要你的输入时是否希望听到通知声音。 默认值是 false。

{
  "agent": {
    "play_sound_when_agent_done": true
  }
}

此设置可通过 Agent 面板的设置 UI 获得。

发送修饰键

使发送消息需要修饰键（在 macOS 上为 cmd，在 Linux 上为 ctrl）。 这鼓励更周到的提示制作。 默认值是 false。

{
  "agent": {
    "use_modifier_to_send": true
  }
}

此设置可通过 Agent 面板的设置 UI 获得。

编辑卡片

使用 expand_edit_card 设置来控制编辑卡片是否在 Agent 面板中显示完整的差异。 默认设置为 true，但如果设置为 false，卡片的高度会被限制为一定的行数，需要点击才能展开。

{
  "agent": {
    "expand_edit_card": false
  }
}

终端卡片

使用 expand_terminal_card 设置来控制终端卡片是否在 Agent 面板中显示命令输出。 默认设置为 true，但如果设置为 false，即使在命令运行时卡片也会完全折叠，需要点击才能展开。

{
  "agent": {
    "expand_terminal_card": false
  }
}

反馈控件

控制是否希望看到赞/踩按钮来给 Zed 反馈关于 agent 的性能。 默认值是 true。

{
  "agent": {
    "enable_feedback": false
  }
}
