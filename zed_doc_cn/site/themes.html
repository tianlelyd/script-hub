主题

Zed 内置了多种主题，还可以通过扩展安装更多主题。

选择主题

通过主题选择器查看已安装的主题并预览它们，你可以通过命令面板中的 "theme selector: Toggle" 打开主题选择器（在 macOS 上绑定为 cmd-k cmd-t，在 Linux 上绑定为 ctrl-k ctrl-t）。

通过上下导航浏览主题列表将实时改变主题，按回车键将把选择保存到你的设置文件中。

安装更多主题

可以从扩展页面获取更多主题，你可以通过命令面板中的 "zed: Extensions" 或 Zed 官方网站访问扩展页面。

许多流行主题已被移植到 Zed，如果你难以选择，可以访问 zed-themes.com，这是一个第三方图库，提供了许多主题的可视化预览。

配置主题

你选择的主题存储在设置文件中。你可以通过命令面板中的 "zed: Open Settings" 打开设置文件（在 macOS 上绑定为 cmd-,，在 Linux 上绑定为 ctrl-,）。

默认情况下，Zed 维护两个主题：一个用于浅色模式，一个用于深色模式。你可以将模式设置为 "dark" 或 "light" 来忽略当前系统模式。

{
  "theme": {
    "mode": "system",
    "light": "One Light",
    "dark": "One Dark"
  }
}

主题覆盖

要覆盖主题的特定属性，请使用 experimental.theme_overrides 设置。

例如，如果你想要覆盖编辑器的背景颜色并将注释和文档注释显示为斜体，可以在你的 settings.json 中添加以下内容：

{
  "experimental.theme_overrides": {
    "editor.background": "#333",
    "syntax": {
      "comment": {
        "font_style": "italic"
      },
      "comment.doc": {
        "font_style": "italic"
      }
    }
  }
}

要查看捕获（如 comment 和 comment.doc）的完整列表，请参阅：语言扩展：语法高亮。

要查看可用主题属性的列表，请查看你主题的 JSON 文件。例如，默认 One Dark 和 One Light 主题的 assets/themes/one/one.json。

本地主题

通过将新主题放置在 ~/.config/zed/themes 目录中来本地存储它们。

例如，要创建一个名为 my-cool-theme 的新主题，在该目录中创建一个名为 my-cool-theme.json 的文件。下次 Zed 加载时，它将在主题选择器中可用。

在 zed-themes.com 查找更多主题。
