外部 Agent

Zed 通过 Agent Client Protocol (ACP) 支持基于终端的 Agent。

目前，Gemini CLI 作为参考实现。 Claude Code 也默认包含在内，您也可以添加自定义的 ACP 兼容 Agent。

Gemini CLI

Zed 提供了在 agent 面板中直接运行 Gemini CLI 的功能。

在幕后，我们在后台运行 Gemini CLI，并通过 ACP 与其进行通信。 这意味着您运行的是真正的 Gemini CLI，拥有其所有优势，但您可以在编辑器中查看并与文件交互。

开始使用

从 Zed Stable v0.201.5 开始，您应该能够直接从 Zed 使用 Gemini CLI。首先使用 cmd-?|ctrl-? 打开 agent 面板，然后使用右上角的 + 按钮启动一个新的 Gemini CLI 线程。

如果您希望将此操作绑定到键盘快捷键，可以通过 zed: open keymap 命令编辑您的 keymap.json 文件，包含：

[
  {
    "bindings": {
      "cmd-alt-g": ["agent::NewExternalAgentThread", { "agent": "gemini" }]
    }
  }
]

安装

第一次创建 Gemini CLI 线程时，Zed 将安装 @google/gemini-cli。此安装仅对 Zed 可用，并会在您使用 agent 时保持更新。

默认情况下，即使您在全局安装了 Gemini CLI，Zed 仍将使用这个管理版本。然而，您可以通过在设置中添加以下内容来配置其使用 PATH 中的版本：

{
  "agent_servers": {
    "gemini": {
      "ignore_system_version": false
    }
  }
}

身份验证

在 Gemini CLI 运行后，系统会提示您选择身份验证方法。

大多数用户应该点击“使用 Google 登录”。这将会弹出一个浏览器窗口，直接与 Gemini CLI 进行身份验证。在这种情况下，Zed 不会看到您的 OAuth 或访问令牌。

您也可以使用“Gemini API Key”。如果您选择这个选项，并已设置了 GEMINI_API_KEY，那么我们将使用该密钥。否则，Zed 将提示您输入 API 密钥，该密钥将安全地存储在您的钥匙串中，用于从 Zed 内启动 Gemini CLI。

“Vertex AI” 选项适用于使用 Vertex AI 并已正确配置环境的用户。

有关更多信息，请参阅 Gemini CLI 文档。

使用方法

与 Zed 的第一方 agent 类似，您可以使用 Gemini CLI 做任何您需要的事情。 为了给它提供上下文，您可以 @-提及文件、最近的线程、符号，或获取 Web 内容。

请注意，一些第一方 agent 功能在 Gemini CLI 上尚不可用：编辑过往消息、从历史中恢复线程、检查点功能以及在 SSH 项目中使用 agent。 我们希望在不久的将来添加这些功能。

Claude Code

与 Gemini CLI 类似，您也可以通过 Zed 的 agent 面板直接运行 Claude Code。 在幕后，Zed 运行 Claude Code 并通过 专用适配器通过 ACP 与其进行通信。

开始使用

使用 cmd-?|ctrl-? 打开 agent 面板，然后使用右上角的 + 按钮启动一个新的 Claude Code 线程。

如果您希望将此操作绑定到键盘快捷键，可以通过 zed: open keymap 命令编辑您的 keymap.json 文件，包含：

[
  {
    "bindings": {
      "cmd-alt-c": ["agent::NewExternalAgentThread", { "agent": "claude_code" }]
    }
  }
]

身份验证

从版本 0.202.7（稳定版）和 0.203.2（预览版）开始，Zed 的 Claude Code 安装的身份验证完全与 Zed 的 agent 分离。也就是说，通过 Zed Agent 设置添加的 Anthropic API 密钥将不会被 Claude Code 用于身份验证和计费。

为了确保您使用所选的计费方式，请打开一个新的 Claude Code 线程。然后，运行 /login，并通过 API 密钥进行身份验证，或通过 使用 Claude Code 登录使用 Claude Pro/Max 订阅。

安装

第一次创建 Claude Code 线程时，Zed 将安装 @zed-industries/claude-code-acp。此安装仅对 Zed 可用，并会在您使用 agent 时保持更新。

即使您在全局安装了 Claude Code，Zed 也将始终使用这个管理版本。

使用方法

与 Zed 的第一方 agent 类似，您可以使用 Claude Code 做任何您需要的事情。 为了给它提供上下文，您可以 @-提及文件、最近的线程、符号，或获取 Web 内容。

除了通过 ACP 进行通信外，Zed 还依赖 Claude Code SDK 来支持其一些特定功能。 然而，SDK 尚未暴露完全支持所有功能所需的一切：

  • Slash 命令：支持 内置命令的一个子集，而 自定义 slash 命令得到完全支持。
  • 子代理得到支持。
  • 钩子目前不支持。

请也注意，一些 第一方 agent 功能在 Claude Code 上尚不可用：编辑过往消息、从历史中恢复线程、检查点功能以及在 SSH 项目中使用 agent。 我们希望在不久的将来添加这些功能。

CLAUDE.md

Zed 中的 Claude Code 将自动使用在项目根目录、项目子目录或根 .claude 目录中找到的任何 CLAUDE.md 文件。

如果您没有 CLAUDE.md 文件，您可以要求 Claude Code 通过 init slash 命令为您创建一个。

添加自定义 Agent

您可以通过以下方式修改设置来在 Zed 中运行任何使用 ACP 的 agent：

{
  "agent_servers": {
    "Custom Agent": {
      "command": "node",
      "args": ["~/projects/agent/index.js", "--acp"],
      "env": {}
    }
  }
}

如果您正在开发一个使用该协议的新 agent 并希望调试它，这也非常有用。

您也可以通过使用 claude 和 gemini 名称为内置集成指定自定义路径、参数或环境。

调试 Agent

在 Zed 中使用外部 agent 时，您可以通过命令面板中的 dev: open acp logs 访问调试视图。这让您可以看到 Zed 和 agent 之间发送和接收的消息。
