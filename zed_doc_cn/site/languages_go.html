Go

Zed 原生支持 Go 语言。

  • 语法解析器: tree-sitter/tree-sitter-go
  • 语言服务器: golang/tools/tree/master/gopls
  • 调试适配器: delve

设置

我们建议通过 Go 的包管理器安装 gopls，而不是通过 Homebrew 或 Linux 发行版的包管理器。

  1. 确保已卸载通过包管理器安装的任何版本的 gopls：
# MacOS homebrew
brew remove gopls
# Ubuntu
sudo apt-get remove gopls
sudo snap remove gopls
# Arch
sudo pacman -R gopls
  1. 使用 Go 模块工具安装/更新 gopls 到最新版本：
go install golang.org/x/tools/gopls@latest
  1. 确保 gopls 在系统路径中：
which gopls
gopls version

如果找不到 gopls，你可能需要将 export PATH="$PATH:$HOME/go/bin" 添加到 .zshrc / .bash_profile 中

内联提示

Zed 为内联提示设置了以下初始化选项：

"hints": {
    "assignVariableTypes": true,
    "compositeLiteralFields": true,
    "compositeLiteralTypes": true,
    "constantValues": true,
    "functionTypeParameters": true,
    "parameterNames": true,
    "rangeVariableTypes": true
}

以便在 Zed 设置中启用内联提示时让语言服务器返回内联提示。

使用以下配置

"lsp": {
    "gopls": {
        "initialization_options": {
            "hints": {
                ....
            }
        }
    }
}

来覆盖这些设置。

更多信息请参见 gopls 内联提示文档。

调试

Zed 支持零配置调试 Go 测试和入口点 (func main)。运行 debugger: start (f4|f4) 可查看这些预配置调试任务的上下文列表。

如需更多控制，可以将调试配置添加到 .zed/debug.json。请参见下面的示例。

调试 Go 包

要调试特定包，可以将 Delve 模式设置为 "debug"。在这种情况下，"program" 应设置为包名称。

[
  {
    "label": "Go (Delve)",
    "adapter": "Delve",
    "program": "$ZED_FILE",
    "request": "launch",
    "mode": "debug"
  },
  {
    "label": "Run server",
    "adapter": "Delve",
    "request": "launch",
    "mode": "debug",
    // 对于 Delve，程序可以是包名称
    "program": "./cmd/server"
    // "args": [],
    // "buildFlags": [],
  }
]

调试 Go 测试

要调试包的测试，将 Delve 模式设置为 "test"。 "program" 仍然是包名称，可以使用 "buildFlags" 来设置标签等操作，使用 "args" 来设置测试二进制文件的参数。(有关详细信息，请参见 go help testflags)。

[
  {
    "label": "Run integration tests",
    "adapter": "Delve",
    "request": "launch",
    "mode": "test",
    "program": ".",
    "buildFlags": ["-tags", "integration"]
    // 要过滤到光标所在的测试：
    // "args": ["-test.run", "$ZED_SYMBOL"]
  }
]

分别构建和调试

如果需要使用特定命令构建应用程序，可以使用 Delve 的 "exec" 模式。在这种情况下，"program" 应指向可执行文件， "build" 命令应构建该文件。

[
  {
    "label": "Debug Prebuilt Unit Tests",
    "adapter": "Delve",
    "request": "launch",
    "mode": "exec",
    "program": "${ZED_WORKTREE_ROOT}/__debug_unit",
    "args": ["-test.v", "-test.run=${ZED_SYMBOL}"],
    "build": {
      "command": "go",
      "args": [
        "test",
        "-c",
        "-tags",
        "unit",
        "-gcflags\"all=-N -l\"",
        "-o",
        "__debug_unit",
        "./pkg/..."
      ]
    }
  }
]

连接到现有的 Delve 实例

您可能需要连接到不一定在本机运行的现有 Delve 实例；在这种情况下，可以使用 tcp_arguments 来配置 Zed 与 Delve 的连接。

[
  {
    "adapter": "Delve",
    "label": "Connect to a running Delve instance",
    "program": "/Users/zed/Projects/language_repositories/golang/hello/hello",
    "cwd": "/Users/zed/Projects/language_repositories/golang/hello",
    "args": [],
    "env": {},
    "request": "launch",
    "mode": "exec",
    "stopOnEntry": false,
    "tcp_connection": { "host": "123.456.789.012", "port": 53412 }
  }
]

在这种情况下，Zed 不会生成新的 Delve 实例，因为它选择使用现有实例。这样做的后果是 Zed 中没有终端；您必须直接与 Delve 实例交互，因为它处理调试程序的 stdin/stdout。
