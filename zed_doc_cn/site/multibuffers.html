多缓冲区

Zed 给你的超能力之一是能够同时编辑多个文件。当与多光标结合使用时，这使得大范围重构变得更加快速。

在多缓冲区中编辑

编辑多缓冲区与编辑普通文件相同。你所做的更改将反映在编辑器中该文件的其他打开副本中，你可以使用 editor: Save 保存所有文件（在 macOS 上绑定到 cmd-s，在 Windows/Linux 上绑定到 ctrl-s，在 Vim 模式下为 :w）。

在多缓冲区中，使用多光标同时编辑每个文件通常很有用。如果你想编辑几个实例，你可以用鼠标选择它们（在 macOS 上 option-点击，在 Windows/Linux 上 alt-点击）或键盘。cmd-d（macOS）、ctrl-d（Windows/Linux）或在 Vim 模式下 gl 将选择光标下单词的下一个匹配项。

当你想编辑所有匹配项时，可以运行 editor: Select All Matches 命令来选择它们（在 macOS 上 cmd-shift-l，在 Windows/Linux 上 ctrl-shift-l，在 Vim 模式下 g a）。

导航到源文件

虽然你可以轻松地在多缓冲区中编辑文件，但直接导航到源文件通常是有益的。你可以通过点击摘录之间的任何分隔线或将光标放在摘录中并执行 editor: open excerpts 命令来实现这一点。需要注意的是，如果使用多个光标，该命令将打开多缓冲区中每个光标下的源文件。

另外，如果你更喜欢使用鼠标并希望双击摘录来打开它，你可以通过设置 "double_click_in_multibuffer": "open" 来启用此功能。

项目搜索

要开始搜索，请运行 pane: Toggle Search 命令（在 macOS 上 cmd-shift-f，在 Windows/Linux 上 ctrl-shift-f，在 Vim 模式下 g/）。搜索完成后，结果将显示在新的多缓冲区中。整个项目中每个匹配行都会有一个摘录。

诊断

如果你已安装语言服务器，诊断面板可以显示项目中的所有错误。你可以通过点击状态栏中的图标或运行 diagnostics: Deploy 命令来打开它（在 macOS 上 cmd-shift-m，在 Windows/Linux 上 ctrl-shift-m，在 Vim 模式下 :clist）。

查找引用

如果你已安装语言服务器，可以使用 editor: Find References 命令查找光标下符号的所有引用（在 macOS 上 cmd-点击，在 Windows/Linux 上 ctrl-点击，在 Vim 模式下 g A）。

根据你的语言服务器，如果有多个可能的定义，editor: Go To Definition 和 editor: Go To Type Definition 等命令也会打开多缓冲区。
