LLM 提供商

要在 Zed 中使用 AI，您需要至少设置一个大型语言模型提供商。

您可以通过订阅 Zed 的套餐之一，或者使用您已有的支持的提供商 API 密钥来实现。

使用您自己的密钥

如果您已经拥有现有 LLM 提供商的 API 密钥——比如 Anthropic 或 OpenAI——您可以将它们添加到 Zed 中，并免费使用 Agent 面板的全部功能。

要为指定提供商添加现有的 API 密钥，请前往 Agent 面板设置（agent: open settings），找到所需的提供商，将密钥粘贴到输入框中，然后按回车键。

注意：API 密钥不会以明文形式存储在您的 settings.json 中，而是存储在您操作系统的安全凭据存储中。

支持的提供商

以下是所有支持您使用自己 API 密钥的 LLM 提供商：

  • Amazon Bedrock
  • Anthropic
  • DeepSeek
  • GitHub Copilot Chat
  • Google AI
  • LM Studio
  • Mistral
  • Ollama
  • OpenAI
  • OpenAI API Compatible
  • OpenRouter
  • Vercel
  • xAI

Amazon Bedrock

支持与支持流式工具使用的模型一起使用工具。 更多详细信息可以在 Amazon Bedrock 的工具使用文档中找到。

要使用 Amazon Bedrock 的模型，需要 AWS 身份验证。 确保您的凭证设置了以下权限：

  • bedrock:InvokeModelWithResponseStream
  • bedrock:InvokeModel

您的 IAM 策略应如下所示：

{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Effect": "Allow",
      "Action": [
        "bedrock:InvokeModel",
        "bedrock:InvokeModelWithResponseStream"
      ],
      "Resource": "*"
    }
  ]
}

完成后，选择以下两种身份验证方法之一：

通过命名配置文件进行身份验证（推荐）

  1. 确保您已安装 AWS CLI 并使用命名配置文件进行配置
  2. 打开您的 settings.json（zed: open settings）并在 language_models 下包含 bedrock 键，设置如下： 
    {
  "language_models": {
    "bedrock": {
      "authentication_method": "named_profile",
      "region": "your-aws-region",
      "profile": "your-profile-name"
    }
  }
}

通过静态凭证进行身份验证

虽然可以通过 Agent 面板设置 UI 直接输入您的 AWS 访问密钥和密钥来配置，但我们建议使用命名配置文件以获得更好的安全实践。 要执行此操作：

  1. 在 IAM 控制台中创建一个您可以承担的 IAM 用户。
  2. 为该用户创建安全凭证，保存它们并确保安全。
  3. 使用（agent: open settings）打开 Agent 配置，然后转到 Amazon Bedrock 部分
  4. 将步骤 2 中的凭证复制到相应的访问密钥 ID、秘密访问密钥和区域字段中。

跨区域推理

Zed 对 Amazon Bedrock 的实现对所有支持的模型和区域组合都使用跨区域推理。 通过跨区域推理，您可以在多个 AWS 区域中分发流量，实现更高的吞吐量。

例如，如果您从 us-east-1 使用 Claude Sonnet 3.7 Thinking，它可能会在美国各区域中处理，即：us-east-1、us-east-2 或 us-west-2。 跨区域推理请求保留在数据原始驻留地理位置的 AWS 区域内。 例如，在美国内发出的请求保留在美国内的 AWS 区域中。

虽然数据仅存储在源区域中，但在跨区域推理期间，您的输入提示和输出结果可能会移动到源区域之外。 所有数据都将通过 Amazon 的安全网络加密传输。

我们将以尽力而为的方式为每个模型支持跨区域推理，请参阅跨区域推理方法代码。

有关最新的支持区域和模型，请参阅跨区域推理支持的模型和区域。

Anthropic

您可以通过 Agent 面板中的模型下拉菜单选择 Anthropic 模型来使用它们。

  1. 注册 Anthropic 并创建 API 密钥
  2. 确保您的 Anthropic 账户有积分
  3. 打开设置视图（agent: open settings）并转到 Anthropic 部分
  4. 输入您的 Anthropic API 密钥

即使您为 Claude Pro 付费，您仍然需要为额外的积分付费才能通过 API 使用它。

如果定义了 ANTHROPIC_API_KEY 环境变量，Zed 也会使用它。

自定义模型

您可以通过将以下内容添加到您的 Zed settings.json 中来为 Anthropic 提供商添加自定义模型：

{
  "language_models": {
    "anthropic": {
      "available_models": [
        {
          "name": "claude-3-5-sonnet-20240620",
          "display_name": "Sonnet 2024-June",
          "max_tokens": 128000,
          "max_output_tokens": 2560,
          "cache_configuration": {
            "max_cache_anchors": 10,
            "min_total_token": 10000,
            "should_speculate": false
          },
          "tool_override": "some-model-that-supports-toolcalling"
        }
      ]
    }
  }
}

自定义模型将会在 Agent 面板的模型下拉菜单中列出。

您可以通过将模型配置中的模式更改为 thinking 来配置模型使用扩展思维（如果它支持的话），例如：

{
  "name": "claude-sonnet-4-latest",
  "display_name": "claude-sonnet-4-thinking",
  "max_tokens": 200000,
  "mode": {
    "type": "thinking",
    "budget_tokens": 4_096
  }
}

DeepSeek

  1. 访问 DeepSeek 平台并创建 API 密钥
  2. 打开设置视图（agent: open settings）并转到 DeepSeek 部分
  3. 输入您的 DeepSeek API 密钥

DeepSeek API 密钥将保存在您的钥匙串中。

如果定义了 DEEPSEEK_API_KEY 环境变量，Zed 也会使用它。

自定义模型

Zed 代理预配置为使用常用模型（DeepSeek Chat、DeepSeek Reasoner）的最新版本。 如果您希望使用备选模型或自定义 API 端点，可以通过向您的 Zed settings.json 添加以下内容来实现：

{
  "language_models": {
    "deepseek": {
      "api_url": "https://api.deepseek.com",
      "available_models": [
        {
          "name": "deepseek-chat",
          "display_name": "DeepSeek Chat",
          "max_tokens": 64000
        },
        {
          "name": "deepseek-reasoner",
          "display_name": "DeepSeek Reasoner",
          "max_tokens": 64000,
          "max_output_tokens": 4096
        }
      ]
    }
  }
}

自定义模型将列在 Agent 面板的模型下拉菜单中。 如果需要，您也可以修改 api_url 来使用自定义端点。

GitHub Copilot Chat

您可以通过在 Agent 面板的模型下拉菜单中选择 GitHub Copilot Chat 来与 Zed 代理一起使用它。

  1. 打开设置视图（agent: open settings）并转到 GitHub Copilot Chat 部分
  2. 点击 Sign in to use GitHub Copilot，按照模态框中显示的步骤操作。

或者，您可以通过 GH_COPILOT_TOKEN 环境变量提供 OAuth 令牌。

注意：如果您在下拉菜单中看不到特定模型，您可能需要在您的 GitHub Copilot 设置中启用它们。

要在 Zed 中使用 Copilot Enterprise（用于代理和自动补全），您必须按照 配置 GitHub Copilot Enterprise 中的描述配置您的企业端点。

Google AI

您可以通过在 Agent 面板的模型下拉菜单中选择 Gemini 模型来与 Zed 代理一起使用它们。

  1. 转到 Google AI Studio 网站并创建 API 密钥。
  2. 打开设置视图（agent: open settings）并转到 Google AI 部分
  3. 输入您的 Google AI API 密钥并按回车。

Google AI API 密钥将保存在您的钥匙串中。

如果定义了 GEMINI_API_KEY 环境变量，Zed 也会使用它。更多信息请参见 Gemini 文档中的使用 Gemini API 密钥。

自定义模型

默认情况下，Zed 将使用模型的 stable 版本，但您可以使用特定版本的模型，包括实验性模型。您可以通过向模型添加 mode 配置来配置模型使用思考模式（如果它支持的话）。这对于控制推理令牌使用和响应速度很有用。如果未指定，Gemini 会自动选择思考预算。

以下是您可以添加到 Zed settings.json 的自定义 Google AI 模型的示例：

{
  "language_models": {
    "google": {
      "available_models": [
        {
          "name": "gemini-2.5-flash-preview-05-20",
          "display_name": "Gemini 2.5 Flash (Thinking)",
          "max_tokens": 1000000,
          "mode": {
            "type": "thinking",
            "budget_tokens": 24000
          }
        }
      ]
    }
  }
}

自定义模型将列在 Agent 面板的模型下拉菜单中。

LM Studio

  1. 下载并安装最新版本的 LM Studio

  2. 在应用中按 cmd/ctrl-shift-m 并下载至少一个模型（例如 qwen2.5-coder-7b）。或者，您可以通过 LM Studio CLI 获取模型：

    lms get qwen2.5-coder-7b

  3. 通过执行以下命令确保 LM Studio API 服务器正在运行：

    lms server start

提示：设置 LM Studio 作为登录项以自动运行 LM Studio 服务器。

Mistral

  1. 访问 Mistral 平台并创建 API 密钥
  2. 打开配置视图（agent: open settings）并导航到 Mistral 部分
  3. 输入您的 Mistral API 密钥

Mistral API 密钥将保存在您的钥匙串中。

如果定义了 MISTRAL_API_KEY 环境变量，Zed 也会使用它。

自定义模型

Zed 代理预配置了几个 Mistral 模型（codestral-latest、mistral-large-latest、mistral-medium-latest、mistral-small-latest、open-mistral-nemo 和 open-codestral-mamba）。 所有默认模型都支持工具使用。 如果您希望使用备选模型或自定义其参数，可以通过向您的 Zed settings.json 添加以下内容来实现：

{
  "language_models": {
    "mistral": {
      "api_url": "https://api.mistral.ai/v1",
      "available_models": [
        {
          "name": "mistral-tiny-latest",
          "display_name": "Mistral Tiny",
          "max_tokens": 32000,
          "max_output_tokens": 4096,
          "max_completion_tokens": 1024,
          "supports_tools": true,
          "supports_images": false
        }
      ]
    }
  }
}

自定义模型将列在 Agent 面板的模型下拉菜单中。

Ollama

从 ollama.com/download 下载并安装 Ollama（Linux 或 macOS），并使用 ollama --version 确保其正在运行。

  1. 下载可用模型之一，例如 mistral：

    ollama pull mistral

  2. 确保 Ollama 服务器正在运行。您可以通过运行 Ollama.app（macOS）或启动以下命令来启动它：

    ollama serve

  3. 在 Agent 面板中，使用模型下拉菜单选择一个 Ollama 模型。

Ollama 上下文长度

Zed 预配置了最大上下文长度（max_tokens）以匹配常见模型的能力。 Zed 向 Ollama 的 API 请求包含此参数作为 num_ctx 参数，但默认值不超过 16384，因此拥有约 16GB RAM 的用户能够开箱即用地使用大多数模型。

有关完整的默认值集，请参见 ollama.rs 中的 get_max_tokens。

注意：Agent 面板中显示的令牌计数仅为估计值，与模型的原生分词器会有所不同。

根据您的硬件或使用情况，您可能希望通过 settings.json 限制或增加特定模型的上下文长度：

{
  "language_models": {
    "ollama": {
      "api_url": "http://localhost:11434",
      "available_models": [
        {
          "name": "qwen2.5-coder",
          "display_name": "qwen 2.5 coder 32K",
          "max_tokens": 32768,
          "supports_tools": true,
          "supports_thinking": true,
          "supports_images": true
        }
      ]
    }
  }
}

如果您指定的上下文长度对于您的硬件来说太大，Ollama 会记录一个错误。 您可以通过运行以下命令查看这些日志：tail -f ~/.ollama/logs/ollama.log（macOS）或 journalctl -u ollama -f（Linux）。 根据您机器可用的内存，您可能需要将上下文长度调整为较小的值。

您还可以为每个可用模型可选择性地指定 keep_alive 的值。 这可以是一个整数（秒）或者一个字符串持续时间，如 "5m"、"10m"、"1h"、"1d" 等。 例如，"keep_alive": "120s" 将允许远程服务器在 120 秒后卸载模型（释放 GPU VRAM）。

supports_tools 选项控制模型是否将使用额外的工具。 如果模型在 Ollama 目录中标记为 tools，应该提供此选项，内置的 Ask 和 Write 配置文件可以使用。 如果模型在 Ollama 目录中未标记为 tools，此选项仍可以设置为 true；但是，请注意只有 Minimal 内置配置文件会工作。

supports_thinking 选项控制模型是否在产生最终答案之前执行明确的"思考"（推理）过程。 如果模型在 Ollama 目录中标记为 thinking，设置此选项，您就可以在 Zed 中使用它。

supports_images 选项启用模型的视觉能力，允许它处理对话上下文中包含的图像。 如果模型在 Ollama 目录中标记为 vision，设置此选项，您就可以在 Zed 中使用它。

OpenAI

  1. 访问 OpenAI 平台并创建 API 密钥
  2. 确保您的 OpenAI 账户有积分
  3. 打开设置视图（agent: open settings）并转到 OpenAI 部分
  4. 输入您的 OpenAI API 密钥

OpenAI API 密钥将保存在您的钥匙串中。

如果定义了 OPENAI_API_KEY 环境变量，Zed 也会使用它。

自定义模型

Zed 代理预配置为使用常见模型（GPT-5、GPT-5 mini、o4-mini、GPT-4.1 等）的最新版本。 要使用备选模型（可能是预览版本），或者如果您希望控制请求参数，可以通过向您的 Zed settings.json 添加以下内容来实现：

{
  "language_models": {
    "openai": {
      "available_models": [
        {
          "name": "gpt-5",
          "display_name": "gpt-5 high",
          "reasoning_effort": "high",
          "max_tokens": 272000,
          "max_completion_tokens": 20000
        },
        {
          "name": "gpt-4o-2024-08-06",
          "display_name": "GPT 4o Summer 2024",
          "max_tokens": 128000
        }
      ]
    }
  }
}

您必须在 max_tokens 参数中提供模型的上下文窗口；这可以在 OpenAI 模型文档中找到。

OpenAI o1 模型还应该设置 max_completion_tokens 以避免产生高推理令牌成本。 自定义模型将列在 Agent 面板的模型下拉菜单中。

OpenAI API Compatible

Zed 支持通过为 OpenAI 提供商指定自定义 api_url 和 available_models 来使用 OpenAI 兼容的 API。 这对于连接其他托管服务（如 Together AI、Anyscale 等）或本地模型很有用。

您可以通过 UI 或编辑您的 settings.json 来添加自定义的、OpenAI 兼容的模型。

要通过 UI 进行操作，请转到 Agent 面板设置（agent: open settings）并查找"LLM Providers"部分标题右侧的"Add Provider"按钮。 然后，填写模态框中可用的输入字段。

要通过您的 settings.json 进行操作，请在 language_models 下添加以下代码片段：

{
  "language_models": {
    "openai_compatible": {
      // Using Together AI as an example
      "Together AI": {
        "api_url": "https://api.together.xyz/v1",
        "available_models": [
          {
            "name": "mistralai/Mixtral-8x7B-Instruct-v0.1",
            "display_name": "Together Mixtral 8x7B",
            "max_tokens": 32768,
            "capabilities": {
              "tools": true,
              "images": false,
              "parallel_tool_calls": false,
              "prompt_cache_key": false
            }
          }
        ]
      }
    }
  }
}

默认情况下，OpenAI 兼容模型继承以下能力：

  • tools: true（支持工具/函数调用）
  • images: false（不支持图像输入）
  • parallel_tool_calls: false（不支持 parallel_tool_calls 参数）
  • prompt_cache_key: false（不支持 prompt_cache_key 参数）

请注意，LLM API 密钥不存储在您的设置文件中。 因此，请确保在您的环境变量中设置它（<PROVIDER_NAME>_API_KEY=<your api key>），以便您的设置可以获取它。在上面的示例中，应该是 TOGETHER_AI_API_KEY=<your api key>。

OpenRouter

OpenRouter 通过单一 API 提供多个 AI 模型的访问。它支持兼容模型的工具使用。

  1. 访问 OpenRouter 并创建账户
  2. 从您的 OpenRouter 密钥页面生成 API 密钥
  3. 打开设置视图（agent: open settings）并转到 OpenRouter 部分
  4. 输入您的 OpenRouter API 密钥

OpenRouter API 密钥将保存在您的钥匙串中。

如果定义了 OPENROUTER_API_KEY 环境变量，Zed 也会使用它。

自定义模型

您可以通过向您的 Zed settings.json 添加以下内容来向 OpenRouter 提供商添加自定义模型：

{
  "language_models": {
    "open_router": {
      "api_url": "https://openrouter.ai/api/v1",
      "available_models": [
        {
          "name": "google/gemini-2.0-flash-thinking-exp",
          "display_name": "Gemini 2.0 Flash (Thinking)",
          "max_tokens": 200000,
          "max_output_tokens": 8192,
          "supports_tools": true,
          "supports_images": true,
          "mode": {
            "type": "thinking",
            "budget_tokens": 8000
          }
        }
      ]
    }
  }
}

每个模型的可用配置选项包括：

  • name（必需）：OpenRouter 使用的模型标识符
  • display_name（可选）：在 UI 中显示的人类可读名称
  • max_tokens（必需）：模型的上下文窗口大小
  • max_output_tokens（可选）：模型可以生成的最大令牌数
  • max_completion_tokens（可选）：最大完成令牌数
  • supports_tools（可选）：模型是否支持工具/函数调用
  • supports_images（可选）：模型是否支持图像输入
  • mode（可选）：思考模型的特殊模式配置

您可以在 OpenRouter 模型页面上找到可用的模型及其规格。

自定义模型将列在 Agent 面板的模型下拉菜单中。

Vercel v0

Vercel v0 是一个专门用于生成全栈应用的专家模型，具有针对 Next.js 和 Vercel 等现代技术栈优化的框架感知完成功能。 它支持文本和图像输入，并提供快速流式响应。

v0 模型是 OpenAI 兼容模型，但 Vercel 在面板的设置视图中被列为一级提供商。

要开始在 Zed 中使用它，请确保您先创建了 v0 API 密钥。 获得密钥后，将其直接粘贴到面板设置视图中的 Vercel 提供商部分。

然后您应该在 Agent 面板的模型下拉菜单中找到它作为 v0-1.5-md。

xAI

Zed 对 xAI 模型提供一级支持。您可以使用您自己的 API 密钥来访问 Grok 模型。

  1. 在 xAI 控制台中创建 API 密钥
  2. 打开设置视图（agent: open settings）并转到 xAI 部分
  3. 输入您的 xAI API 密钥

xAI API 密钥将保存在您的钥匙串中。如果定义了 XAI_API_KEY 环境变量，Zed 也会使用它。

注意：虽然 xAI API 与 OpenAI 兼容，但 Zed 对其作为专用提供商提供一级支持。为了获得最佳体验，我们建议使用专用的 x_ai 提供商配置而不是 OpenAI API Compatible 方法。

自定义模型

Zed 代理预配置了常见的 Grok 模型。如果您希望使用备选模型或自定义其参数，可以通过向您的 Zed settings.json 添加以下内容来实现：

{
  "language_models": {
    "x_ai": {
      "api_url": "https://api.x.ai/v1",
      "available_models": [
        {
          "name": "grok-1.5",
          "display_name": "Grok 1.5",
          "max_tokens": 131072,
          "max_output_tokens": 8192
        },
        {
          "name": "grok-1.5v",
          "display_name": "Grok 1.5V (Vision)",
          "max_tokens": 131072,
          "max_output_tokens": 8192,
          "supports_images": true
        }
      ]
    }
  }
}

Custom Provider Endpoints

您可以为不同的提供商使用自定义 API 端点，只要它与提供商的 API 结构兼容。 要做到这一点，请向您的 settings.json 添加以下内容：

{
  "language_models": {
    "some-provider": {
      "api_url": "http://localhost:11434"
    }
  }
}

目前，some-provider 可以是以下任何一个值：anthropic、google、ollama、openai。

这是支持模型的相同基础设施，例如支持 OpenAI 兼容模型。
