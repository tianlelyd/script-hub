JSON

JSON 支持在 Zed 中原生可用。

  • Tree-sitter: tree-sitter/tree-sitter-json
  • Language Server: zed-industries/json-language-server

JSONC

Zed 还支持一个 JSON 的超集称为 JSONC，它允许在 JSON 文件中使用单行注释（//）。 在编辑这些文件时，您可以使用 cmd-/（macOS）或 ctrl-/（Linux）来切换当前行或选择内容的注释状态。

JSONC Prettier Formatting

如果您在使用 Format Document 或启用了 format_on_save 时使用扩展名为 *.jsonc 的文件，Zed 会调用 Prettier 作为格式化程序。Prettier 有一个未解决的问题，会在扩展名为 jsonc 的文件中添加尾随逗号。扩展名为 .json 的 JSONC 文件不受影响。

为了解决这个问题，您可以将以下内容添加到您的 .prettierrc 配置文件中：

{
  "overrides": [
    {
      "files": ["*.jsonc"],
      "options": {
        "parser": "json",
        "trailingComma": "none"
      }
    }
  ]
}

JSON Language Server

Zed 开箱即用地自动支持 package.json 和 tsconfig.json 文件的 JSON Schema 验证，但 json-language-server 可以使用项目文件中的 JSON Schema 定义、来自 JSON Schema Store 或其他公开可用的 URL 进行 JSON 验证。

内联 Schema 规范

要在您的 JSON 文件中内联指定 schema，请添加一个链接到您的 json schema 文件的 $schema 顶级键。

例如，为与 lua-language-server 一起使用的 .luarc.json 文件：

{
  "$schema": "https://raw.githubusercontent.com/sumneko/vscode-lua/master/setting/schema.json",
  "runtime.version": "Lua 5.4"
}

通过设置指定 Schema

您也可以通过 Zed LSP 设置将 JSON Schema 与文件路径关联。

要做到这一点：

"lsp": {
  "json-language-server": {
    "settings": {
      "json": {
        "schemas": [
          {
            "fileMatch": ["*/*.luarc.json"],
            "url": "https://raw.githubusercontent.com/sumneko/vscode-lua/master/setting/schema.json"
          }
        ]
      }
    }
  }
}
