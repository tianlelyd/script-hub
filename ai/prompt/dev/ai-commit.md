请根据以下输入的改动内容，生成符合 **Conventional Commits** 规范的 Git 提交信息（使用中文）：  

1. **标题（Summary）**：一句话概述改动，控制在 50-72 个字符内，格式为：  
   `type(scope): 简短描述`  
   其中 `type` 可取：  
   - `feat`（新功能）  
   - `fix`（修复 Bug）  
   - `docs`（文档修改）  
   - `style`（代码格式）  
   - `refactor`（重构）  
   - `test`（测试）  
   - `chore`（构建、脚手架、依赖等杂项）  

2. **正文（Description）**：详细说明改动的原因、影响和上下文。  
   - 每行不超过 72 个字符。  
   - 可以分点列出主要改动。  
   - 如果关联 issue，在末尾添加 `关联: #<issue号>`。  

3. 风格应简洁、明确、专业，避免模糊词汇（如 “更新代码”、“修改东西”）。  

**输入示例：**  
修改了用户登录逻辑，修复了因 token 过期导致的闪退问题，添加了 token 自动刷新机制，并优化了登录界面加载速度。

**输出示例：**  
feat(auth): 增加 token 自动刷新机制，修复登录闪退  

- 实现 token 自动刷新，避免 token 过期导致闪退  
- 优化登录界面加载速度，提升用户体验  
关联: #123
